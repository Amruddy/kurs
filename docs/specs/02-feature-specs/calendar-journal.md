# Feature Spec

# Calendar Journal

## 1. Назначение функции

Calendar Journal - это календарный журнал обучения.

Он нужен, чтобы преподаватель мог вести учебный учет по группе или индивидуальному ученику: занятия, отметки урока, комментарии, прогресс и домашние задания.

Журнал - главный рабочий инструмент MVP. Он должен заменить бумажную тетрадь или таблицу, но остаться быстрым и понятным для преподавателя.

## 2. Для кого функция

### 2.1. Преподаватель

Преподаватель - основной пользователь журнала.

Он может:

- открыть журнал группы;
- открыть журнал индивидуального ученика;
- видеть занятия за выбранный период;
- фиксировать записи по урокам;
- ставить отметку урока;
- оставлять комментарии;
- отмечать прогресс;
- видеть историю ученика.

### 2.2. Преподаватель-одиночка

Преподаватель-одиночка использует журнал так же, как преподаватель.

Разница только в том, что он сам создает курс, группу или индивидуального ученика, расписание и правила оплаты.

### 2.3. Администратор / директор

Администратор/директор может смотреть журнал, но не редактирует учебные записи.

Он использует журнал для контроля:

- идет ли обучение;
- есть ли занятия;
- есть ли пропуски;
- есть ли комментарии по ученикам;
- ведет ли преподаватель учет.

### 2.4. Ученик

Ученик не работает с журналом как преподаватель.

Он может видеть только те данные, которые предназначены для ученика:

- домашнее задание;
- открытые комментарии преподавателя;
- открытый прогресс;
- то, что нужно повторить.

## 3. Главная идея журнала

Журнал должен показывать обучение по датам занятий.

Преподаватель должен быстро понять:

- какие занятия были в этом месяце;
- кто отсутствовал;
- у кого есть уважительная причина;
- кому поставлена отметка или оценка;
- где есть комментарий;
- где есть запись по прогрессу;
- что было задано на дом.

Журнал не должен превращаться в сложную систему отчетов. Главное - быстро открыть, отметить урок и сохранить.

## 4. Что открывает преподаватель

Преподаватель может открыть:

- журнал группы;
- журнал индивидуального ученика.

Журнал группы относится к конкретному курсу, группе и списку учеников.

Журнал индивидуального ученика относится к конкретному курсу и одному ученику.

Индивидуальный ученик не должен называться группой.

## 5. Период журнала

В MVP журнал должен поддерживать просмотр занятий по месяцу.

Преподаватель может:

- выбрать месяц;
- видеть занятия выбранного месяца;
- перейти к предыдущему месяцу;
- перейти к следующему месяцу;
- открыть конкретный урок.

Если курс идет несколько месяцев, журнал должен позволять смотреть каждый месяц отдельно.

## 6. Запись урока

Запись урока - это информация по конкретному ученику на конкретном занятии.

Запись урока может содержать:

- отметку урока;
- комментарий преподавателя;
- комментарий об опоздании;
- краткую запись по прогрессу;
- связь с домашним заданием;
- заметку, видимую только преподавателю;
- заметку, которую можно открыть ученику.

Оценка за каждый урок не обязательна.

Отметка урока ставится в одном месте. Она может означать:

- пусто - ученик был на уроке, если посещаемость подтверждена или урок завершен преподавателем;
- `П` - присутствовал, если преподаватель хочет отметить явно;
- `Н` - отсутствовал;
- `У` - отсутствовал по уважительной причине;
- оценка за урок, если преподаватель ее ставит.

Но запись не должна оставаться непонятной: по уроку должно быть видно, что произошло.

Понятной записью урока считается хотя бы одно из значений:

- отметка урока;
- оценка;
- комментарий;
- прогресс.

## 7. Быстрые обозначения

Для быстрых отметок урока в журнале можно использовать короткие русские обозначения.

Базовые обозначения MVP:

- `П` - присутствовал;
- `Н` - отсутствовал;
- `У` - отсутствовал по уважительной причине.

Опоздание не является отдельным обязательным обозначением. Если ученик опоздал, преподаватель указывает это в комментарии к уроку.

Уровень по правилу таджвида ставится в блоке прогресса, а не как общая оценка урока.

Эти обозначения считаются базово закрепленными для MVP. Детали их поведения уточняются в Feature Spec посещаемости.

## 8. Отметка урока в журнале

Преподаватель может:

- отметить отсутствие;
- отметить уважительную причину;
- явно поставить присутствие;
- убрать ошибочную отметку;
- оставить комментарий к отсутствию;
- указать опоздание в комментарии.

Если ученик присутствовал и проблем нет, отдельная отметка может быть необязательной. Отсутствие какой-либо отметки в ячейке урока означает, что ученик был на уроке, только после подтверждения посещаемости или завершения урока преподавателем. Но в журнале должна быть возможность явно поставить `П`, если преподавателю так удобнее.

## 9. Оценка урока и уровень по правилу

В MVP есть две разные вещи:

- оценка за урок;
- уровень по правилу таджвида, добавленному этому ученику.

Оценка за урок ставится как общая отметка урока.

В MVP шкала оценки за урок хранится в настройках курса (`lesson_mark_scale`). Ее выбирает администратор при создании курса или преподаватель-одиночка для своего курса:

- 5-балльная шкала;
- 10-балльная шкала.

Уровень по правилу ставится отдельно в блоке прогресса. Например, преподаватель добавил ученику правило "Сыфаты" и может поставить этому ученику уровень именно по этому правилу.

Преподаватель может:

- поставить оценку за урок;
- открыть прогресс конкретного ученика;
- выбрать из правил ученика или добавить новое правило ученику;
- поставить уровень по правилу ученика;
- изменить уровень по правилу ученика;
- убрать ошибочный уровень из прогресса ученика;
- оставить комментарий вместо оценки;
- оставить уровень вместе с комментарием.

Для уровня по правилу используется простая качественная шкала:

- отлично;
- хорошо;
- удовлетворительно;
- плохо.

5-балльная и 10-балльная шкалы относятся только к общей оценке за урок, а не к уровню по правилу.

## 10. Комментарии

В журнале нужны два типа комментариев:

- комментарий для преподавателя;
- комментарий, который может видеть ученик.

Комментарий для преподавателя нужен для внутренних заметок:

- ученик опоздал;
- плохо подготовился;
- нужно повторить;
- есть проблема с произношением;
- нужно поговорить с учеником или родителем;
- есть важная деталь по уроку.

Комментарий для ученика нужен, чтобы ученик понял:

- что повторить;
- что улучшить;
- на что обратить внимание;
- какое замечание дал преподаватель.

В MVP преподаватель сам выбирает, какие комментарии и части прогресса видит ученик.

## 11. Прогресс в журнале

Журнал может показывать или открывать простую запись по прогрессу таджвида.

В MVP прогресс не строится как большой фиксированный шаблон категорий.

Преподаватель открывает прогресс конкретного ученика и работает с:

- правилами таджвида, добавленными этому ученику;
- ошибками чтения, добавленными этому ученику;
- что нужно повторить;
- комментарий;
- видимость для ученика.

Для уровня по правилу используются:

- отлично;
- хорошо;
- удовлетворительно;
- плохо;

В журнале не нужно показывать длинные списки правил и ошибок прямо в ячейке. Главное, чтобы преподаватель мог открыть ученика или урок и внести нужные данные по прогрессу.

## 12. Домашнее задание в журнале

Преподаватель может связать урок с домашним заданием.

Домашнее задание в MVP:

- создается преподавателем;
- содержит текст задания;
- может иметь срок;
- может иметь комментарий;
- может иметь текстовый материал или ссылку;
- видно ученику.

Ученик не отправляет выполненное домашнее задание через систему.

Домашнее задание нужно для фиксации: преподаватель указал, ученик увидел, дальше ученик сдает его вживую или на уроке.

## 13. Материалы в журнале

Преподаватель может связать урок с материалом.

В MVP материал - это:

- текстовое описание;
- текстовый материал;
- ссылка.

Загрузка файлов, PDF, аудио, видео и других медиа не входит в MVP.

## 14. Права доступа

### 14.1. Преподаватель

Преподаватель может редактировать журнал своих групп и индивидуальных учеников.

Он может:

- добавлять записи;
- изменять записи;
- удалять ошибочные значения;
- писать комментарии;
- управлять видимостью комментариев для ученика.

### 14.2. Администратор / директор

Администратор/директор может смотреть журнал.

Он не может редактировать учебные записи журнала в MVP.

### 14.3. Ученик

Ученик не редактирует журнал.

Он видит только открытые для него данные:

- домашнее задание;
- комментарии для ученика;
- открытый прогресс;
- открытые правила, ошибки и комментарии по прогрессу.

## 15. Основные пользовательские пути

### 15.1. Преподаватель отмечает урок группы

1. Преподаватель открывает курс.
2. Выбирает группу.
3. Открывает журнал.
4. Выбирает нужный месяц.
5. Открывает урок.
6. Отмечает отсутствующих.
7. При необходимости отмечает уважительную причину.
8. При необходимости ставит оценку в отметке урока.
9. Добавляет комментарии.
10. При необходимости добавляет запись по прогрессу.
11. При необходимости задает домашнее задание.
12. Сохраняет запись.

### 15.2. Преподаватель ведет индивидуального ученика

1. Преподаватель открывает курс.
2. Выбирает индивидуального ученика.
3. Открывает журнал индивидуального обучения.
4. Выбирает урок.
5. Вносит оценку, комментарий, прогресс или домашнее задание.
6. Сохраняет запись.

### 15.3. Администратор/директор смотрит журнал

1. Администратор/директор открывает группу.
2. Открывает журнал.
3. Смотрит занятия и записи.
4. Проверяет, ведется ли учет.
5. Не редактирует учебные записи.

### 15.4. Ученик получает данные из журнала

1. Ученик открывает личный кабинет.
2. Видит домашнее задание.
3. Видит открытые комментарии.
4. Видит открытый прогресс: правила, уровни, ошибки или комментарии, которые преподаватель открыл ученику.
5. Не редактирует журнал.

## 16. Проблемные ситуации

### 16.1. Ученик отсутствовал

Преподаватель должен быстро отметить отсутствие.

Если причина уважительная, преподаватель может отметить уважительную причину.

### 16.2. Ученик опоздал

Опоздание фиксируется комментарием к уроку.

Отдельный обязательный статус для опоздания не нужен в MVP.

### 16.3. Преподаватель ошибся

Преподаватель должен иметь возможность исправить ошибочную отметку, оценку или комментарий.

### 16.4. Урок прошел, но оценки нет

Это допустимо.

Оценка не обязательна для каждого урока. Вместо оценки может быть комментарий или другая понятная запись.

### 16.5. Ученик не должен видеть внутреннюю заметку

Преподаватель должен иметь возможность оставить заметку только для себя.

Такая заметка не должна показываться ученику.

## 17. Перенос урока

Если урок переносится, система должна сохранить связь между исходной датой и новой датой.

В расписании:

- исходный урок помечается как перенесен;
- указывается новая дата и время;
- сохраняется связь с исходным уроком.

В журнале:

- не создается учебная запись задним числом на старую дату;
- журнал заполняется только по фактически проведенному занятию;
- на старой дате показывается статус "перенесен";
- на новой дате преподаватель получает обычную возможность отметить тему, посещаемость, оценку, комментарий, прогресс и домашнее задание.

## 18. Что входит в MVP

В MVP входит:

- журнал группы;
- журнал индивидуального ученика;
- просмотр по месяцу;
- записи по датам занятий;
- единая отметка урока;
- отметка отсутствия;
- отметка уважительной причины;
- опоздание как комментарий;
- оценка урока как вариант отметки урока;
- комментарий преподавателя;
- комментарий для ученика;
- связь с прогрессом таджвида;
- связь с домашним заданием;
- связь с текстовым материалом или ссылкой;
- просмотр журнала администратором/директором без редактирования;
- просмотр открытых данных учеником.
- перенос урока со связью исходной и новой даты.

## 19. Что не входит в MVP

В MVP не входит:

- загрузка файлов;
- PDF;
- аудио;
- видео;
- автоматическая проверка домашнего задания;
- сдача домашнего задания через систему;
- редактирование журнала учеником;
- редактирование журнала администратором/директором по обычной роли без отдельного аварийного права;
- сложная аналитика;
- автоматические уведомления.

## 20. Открытые вопросы для следующих спецификаций

- как журнал связан с повторяющимся расписанием;
- как журнал должен вести себя при изменении состава группы.
- нужна ли после MVP отдельная шкала оценки на уровне группы или преподавателя.
