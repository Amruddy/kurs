# Feature Spec

# Admin Dashboard

## 1. Назначение функции

Admin Dashboard - это рабочая область администратора или директора.

Функция нужна, чтобы видеть состояние школы: курсы, группы, преподавателей, учеников, расписание, ведение журнала, посещаемость, прогресс и оплату.

В MPMF 1.0 админ-панель должна помогать управлять учебным процессом, а не строить сложную аналитику.

## 2. Для кого функция

### 2.1. Администратор / директор

Администратор/директор является основным пользователем.

Он может:

- видеть список курсов;
- видеть список групп;
- видеть список преподавателей;
- видеть список учеников;
- видеть расписание;
- видеть журналы;
- видеть посещаемость;
- видеть прогресс;
- видеть оплату;
- видеть проблемные группы и учеников.

### 2.2. Преподаватель

Преподаватель не использует админ-панель школы, если у него нет административных прав.

### 2.3. Преподаватель-одиночка

Преподаватель-одиночка получает упрощенную административную область для своих курсов, групп, учеников и оплат.

### 2.4. Ученик

Ученик не имеет доступа к админ-панели.

## 3. Главная идея

Администратор должен быстро понять:

- какие курсы есть;
- какие группы активны;
- где нет расписания;
- где нет преподавателя;
- сколько учеников учится;
- кто часто пропускает;
- у кого слабый прогресс;
- кто не оплатил.

## 4. Главный экран

Главный экран админ-панели показывает краткую картину:

- количество курсов;
- количество групп;
- количество преподавателей;
- количество учеников;
- активные группы;
- группы без расписания;
- группы без преподавателя;
- ученики с частыми пропусками;
- ученики с задолженностью;
- последние изменения в журналах.

На главном экране также должны быть видимые быстрые действия для основных
созданий: добавить курс, добавить группу, добавить ученика и добавить
преподавателя. Эти действия не должны быть спрятаны ниже длинных списков,
чтобы администратор мог начать работу сразу после входа.

## 5. Курсы

Администратор может:

- создать курс;
- открыть курс;
- изменить курс;
- видеть группы курса;
- видеть индивидуальных учеников курса;
- архивировать курс.

## 6. Группы

Администратор может:

- создать группу;
- выбрать курс;
- назначить преподавателя;
- добавить учеников;
- изменить состав;
- изменить расписание;
- изменить статус;
- открыть журнал группы;
- видеть оплату по группе.

## 7. Преподаватели

Администратор может:

- видеть список преподавателей;
- открыть преподавателя;
- назначить преподавателя на группу;
- видеть группы преподавателя;
- видеть индивидуальных учеников преподавателя;
- отключить преподавателя от группы.

В MPMF 1.0 не требуется сложный HR-модуль преподавателей.

## 8. Ученики

Администратор может:

- добавить ученика;
- открыть карточку ученика;
- назначить ученика в группу;
- назначить ученика на индивидуальное обучение;
- видеть посещаемость;
- видеть прогресс;
- видеть оплату;
- изменить контактные данные;
- архивировать ученика.

## 9. Контроль журнала

Администратор может смотреть журналы групп и индивидуальных учеников.

В MPMF 1.0 администратор не редактирует учебные записи преподавателя.

Он может проверить:

- есть ли занятия;
- ведется ли посещаемость;
- есть ли домашние задания;
- есть ли комментарии;
- фиксируется ли прогресс.

## 10. Контроль посещаемости и прогресса

Администратор может видеть:

- учеников с частыми пропусками;
- группы с проблемной посещаемостью;
- учеников со слабым прогрессом;
- историю прогресса ученика.

Это контрольные признаки, а не сложная аналитика.

## 11. Контроль оплаты

Администратор может видеть:

- оплату по всем группам;
- оплату по конкретной группе;
- оплату по конкретному ученику;
- список неоплативших;
- сроки оплаты;
- статусы оплаты.

Он может вручную менять статусы оплаты.

## 12. Пользовательские пути

### 12.1. Администратор проверяет проблемные группы

1. Администратор открывает админ-панель.
2. Видит группы без расписания или преподавателя.
3. Открывает группу.
4. Исправляет расписание или назначает преподавателя.

### 12.2. Администратор проверяет неоплативших

1. Администратор открывает оплату.
2. Фильтрует задолженности.
3. Видит учеников и сроки.
4. После получения оплаты меняет статус вручную.

### 12.3. Администратор проверяет ведение журнала

1. Администратор открывает группу.
2. Открывает журнал.
3. Проверяет наличие занятий, отметок, заданий и прогресса.

## 13. Проблемные ситуации

### 13.1. Группа без расписания

Админ-панель должна показывать такую группу как проблемную.

### 13.2. Группа без преподавателя

Админ-панель должна показывать такую группу как требующую назначения преподавателя.

### 13.3. Ученик часто пропускает

Администратор видит признак частых пропусков и может открыть карточку ученика.

### 13.4. Ученик не оплатил

Администратор видит ученика в списке неоплативших.

## 14. Права доступа

### 14.1. Администратор / директор

Имеет доступ к админ-панели школы.

### 14.2. Преподаватель-одиночка

Имеет доступ к упрощенной административной области своих данных.

### 14.3. Преподаватель

Не имеет доступа к админ-панели без дополнительных прав.

### 14.4. Ученик

Не имеет доступа к админ-панели.

## 15. Что входит в MPMF 1.0

В MPMF 1.0 входит:

- обзор школы;
- курсы;
- группы;
- преподаватели;
- ученики;
- расписание;
- просмотр журналов;
- контроль посещаемости;
- контроль прогресса;
- ручной учет оплат;
- проблемные признаки.

## 16. Что не входит в MPMF 1.0

В MPMF 1.0 не входит:

- сложная аналитика;
- финансовая бухгалтерия;
- HR-модуль;
- автоматические уведомления;
- отчеты для печати;
- экспорт в Excel;
- роли с гибкой матрицей прав.

## 17. Открытые вопросы для следующих спецификаций

- Какие метрики должны быть на главном экране первыми.
- Нужен ли отдельный директорский режим после MPMF 1.0.
- Как разделить права администратора и директора технически.
