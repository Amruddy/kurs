# Public Homepage

# Публичная главная страница проекта

## 1. Назначение

Публичная главная страница нужна как лицо проекта до входа в систему.

Она должна быстро объяснять посетителю:

- что такое `Курс`;
- для кого создан продукт;
- какую учебную задачу он решает;
- какие возможности уже входят в MVP;
- как перейти ко входу в систему.

## 2. Основной маршрут

Маршрут:

- `/`

В финальном MVP этот маршрут должен открывать публичную главную страницу.
Текущий временный редирект с `/` на `/login` допустим только до реализации
этого этапа.

## 3. Содержание MVP

Главная страница показывает только то, что соответствует текущему MVP:

- школа или учебный центр может вести курсы, группы, преподавателей и учеников;
- преподаватель может видеть группы, уроки, журнал, посещаемость, прогресс,
  домашние задания и материалы;
- ученик может видеть расписание, домашние задания, материалы, прогресс,
  посещаемость и оплату;
- администратор может контролировать учебную основу и ручной учет оплаты.

## 4. Основные блоки

Страница может содержать:

- hero-блок с названием продукта и коротким описанием;
- блок "для кого";
- блок основных возможностей MVP;
- короткое объяснение про таджвид и образовательный контекст;
- кнопку или ссылку "Войти в систему", ведущую на `/login`.

## 5. Не входит в MVP

На публичной главной странице не нужно добавлять:

- регистрацию;
- тарифы;
- онлайн-оплату;
- форму заявки;
- блог;
- чат;
- отзывы;
- интеграции;
- новые роли;
- новые личные кабинеты;
- функциональность, которой нет в текущем MVP.

## 6. Дизайн

Публичная главная страница должна быть частью той же визуальной системы, что и
рабочий интерфейс:

- светлый образовательный стиль;
- аккуратные панели и карточки;
- мягкие тени;
- понятная типографика;
- спокойные акценты;
- хорошая mobile-адаптация.

Она может быть более презентационной, чем рабочие страницы, но не должна
выглядеть как отдельный чужой маркетинговый сайт.

## 7. Проверка готовности

Готово, если:

- `/` открывается без авторизации;
- `/login` доступен с главной страницы;
- dev-вход админа, преподавателя и ученика продолжает работать;
- страница не добавляет новых действий вне MVP;
- страница визуально совпадает с общим дизайном проекта;
- `npm.cmd run lint`, `npm.cmd run build` и `npm.cmd run smoke:mvp` проходят.
