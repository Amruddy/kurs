# Feature Spec

# Tajweed Progress

## 1. Назначение функции

Tajweed Progress - это учет прогресса ученика по таджвиду.

Функция нужна, чтобы преподаватель мог фиксировать не просто оценку за урок, а состояние чтения ученика: сильные стороны, проблемные зоны, что нужно повторить и какие цели стоят перед учеником.

В MVP прогресс строится вокруг готового шаблона таджвида. Полноценный конструктор любых предметов не входит в MVP.

## 2. Для кого функция

### 2.1. Преподаватель

Преподаватель - основной пользователь прогресса.

Он может:

- видеть категории прогресса таджвида;
- ставить уровень по категории;
- менять уровень;
- оставлять комментарии;
- отмечать проблемные зоны;
- указывать, что нужно повторить;
- задавать индивидуальную цель;
- выбирать, что видит ученик.

### 2.2. Преподаватель-одиночка

Преподаватель-одиночка использует прогресс так же, как преподаватель.

Он сам создает курс, группу или индивидуального ученика и сам ведет прогресс.

### 2.3. Администратор / директор

Администратор/директор может смотреть прогресс, но не редактирует учебные записи в MVP.

Он использует прогресс для контроля качества обучения.

### 2.4. Ученик

Ученик видит только те части прогресса, которые преподаватель открыл для ученика.

Ученик не редактирует прогресс.

## 3. Главная идея

Прогресс таджвида должен помогать преподавателю и ученику понимать:

- как ученик читает;
- какие ошибки повторяются;
- какие темы нужно повторить;
- что стало лучше;
- какая цель стоит перед учеником;
- что ученик должен подготовить к следующим урокам.

Прогресс не должен выглядеть как обычный школьный дневник с грубыми оценками. Для таджвида важнее видеть состояние навыка и движение ученика.

## 4. Базовые категории MVP

В MVP шаблон таджвида содержит категории:

- уровень чтения;
- произношение букв;
- махрадж;
- сыфаты;
- мадды;
- правила нун сакина и танвина;
- теория;
- практика;
- ошибки чтения;
- что нужно повторить;
- индивидуальная цель;
- общий комментарий.

Эти категории являются стартовым шаблоном. Преподаватель может минимально редактировать или добавлять критерии внутри шаблона таджвида.

## 5. Уровни по категориям

Для категорий, где нужен уровень, используются простые значения:

- хорошо;
- средне;
- плохо;
- не оценено.

Не каждая категория обязана иметь уровень.

Например:

- `махрадж` может иметь уровень;
- `сыфаты` может иметь уровень;
- `теория` может иметь уровень;
- `ошибки чтения` может быть текстовой записью;
- `что нужно повторить` может быть текстовой записью;
- `индивидуальная цель` может быть текстовой записью.

## 6. Комментарии

Прогресс должен поддерживать комментарии.

Комментарии могут быть:

- внутренние, только для преподавателя;
- открытые для ученика.

Внутренний комментарий нужен для заметок преподавателя.

Открытый комментарий нужен, чтобы ученик понял, что улучшить и что повторить.

Преподаватель сам выбирает, какие комментарии видит ученик.

## 7. Видимость для ученика

В MVP прогресс имеет разные уровни видимости:

- часть прогресса видит только преподаватель;
- часть прогресса может быть открыта ученику;
- ученик видит общий прогресс и комментарии, которые преподаватель считает нужным открыть.

По умолчанию ученик не обязан видеть все категории.

Преподаватель должен контролировать, что именно показывается ученику.

## 8. Связь с журналом

Прогресс связан с календарным журналом.

Преподаватель может:

- открыть урок;
- добавить краткую запись по прогрессу;
- открыть подробный прогресс ученика;
- изменить уровень по категории;
- оставить комментарий;
- указать, что нужно повторить.

Не обязательно заполнять все категории на каждом уроке.

## 9. Связь с учеником

Прогресс хранится по конкретному ученику.

Ученик может быть:

- в группе;
- индивидуальным учеником.

Индивидуальный ученик не должен называться группой.

История прогресса должна сохраняться так, чтобы преподаватель мог видеть движение ученика со временем.

## 10. Связь с курсом

Категории прогресса берутся из курса.

В MVP главный шаблон курса - "Таджвид".

Курс может иметь:

- базовые категории прогресса;
- уровни;
- текстовые поля;
- минимальные настройки критериев.

Полноценный конструктор любых предметов не входит в MVP.

## 11. Пользовательские пути

### 11.1. Преподаватель отмечает прогресс после урока

1. Преподаватель открывает журнал.
2. Выбирает урок.
3. Выбирает ученика.
4. Открывает прогресс.
5. Выбирает нужную категорию.
6. Ставит уровень или пишет комментарий.
7. Указывает, что нужно повторить.
8. При необходимости открывает часть комментария ученику.
9. Сохраняет запись.

### 11.2. Преподаватель смотрит историю прогресса

1. Преподаватель открывает ученика.
2. Открывает прогресс.
3. Смотрит категории.
4. Смотрит комментарии и изменения по времени.
5. Понимает, что улучшилось и что остается проблемой.

### 11.3. Ученик смотрит открытый прогресс

1. Ученик открывает личный кабинет.
2. Видит общий прогресс.
3. Видит открытые преподавателем комментарии.
4. Видит, что нужно повторить.
5. Не видит внутренние заметки преподавателя.

### 11.4. Администратор/директор смотрит прогресс

1. Администратор/директор открывает ученика или группу.
2. Смотрит прогресс.
3. Проверяет, ведется ли учет.
4. Не редактирует учебные записи.

## 12. Проблемные ситуации

### 12.1. Ученик плохо продвигается

Преподаватель может:

- поставить уровень `плохо` или `средне`;
- указать проблемную категорию;
- написать комментарий;
- указать, что нужно повторить;
- задать индивидуальную цель.

### 12.2. Преподаватель не хочет показывать заметку ученику

Преподаватель может оставить внутренний комментарий.

Такой комментарий не показывается ученику.

### 12.3. Категория не нужна конкретному преподавателю

Преподаватель может не использовать категорию.

В MVP не нужно заставлять преподавателя заполнять все категории.

### 12.4. Нужно добавить свой критерий

В MVP допускается минимальное добавление или редактирование критериев внутри шаблона таджвида.

Это не полноценный конструктор предметов.

## 13. Права доступа

### 13.1. Преподаватель

Преподаватель может редактировать прогресс своих групп и индивидуальных учеников.

### 13.2. Администратор / директор

Администратор/директор может смотреть прогресс.

В MVP он не редактирует прогресс.

### 13.3. Ученик

Ученик видит только открытые данные.

Ученик не редактирует прогресс.

## 14. Что входит в MVP

В MVP входит:

- шаблон прогресса таджвида;
- категории прогресса;
- уровни `хорошо`, `средне`, `плохо`, `не оценено`;
- текстовые комментарии;
- внутренние комментарии преподавателя;
- комментарии, открытые ученику;
- отметка проблемных зон;
- поле "что нужно повторить";
- индивидуальная цель;
- связь с календарным журналом;
- история прогресса ученика;
- просмотр открытого прогресса учеником;
- просмотр прогресса администратором/директором без редактирования;
- минимальное редактирование или добавление критериев внутри шаблона таджвида.

## 15. Что не входит в MVP

В MVP не входит:

- полноценный конструктор любых предметов;
- автоматическая оценка чтения;
- автоматическая проверка аудио;
- загрузка аудио;
- сложная аналитика прогресса;
- графики и отчеты продвинутого уровня;
- редактирование прогресса учеником;
- редактирование прогресса администратором/директором.

## 16. Открытые вопросы для следующих спецификаций

- какие категории видит ученик по умолчанию;
- где хранится настройка видимости категории;
- какие категории можно редактировать в шаблоне таджвида;
- можно ли отключать категорию для конкретной группы;
- как показывать историю изменений прогресса;
- как прогресс связан с домашним заданием.
