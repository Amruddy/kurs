# Feature Spec

# Tajweed Progress

## 1. Назначение функции

Tajweed Progress - это простой учет состояния чтения ученика по таджвиду.

В MPMF 1.0 прогресс не должен быть тяжелой системой рубрик, категорий и сложных оценок.

Главная идея: преподаватель в карточке ученика видит состояние чтения по тем правилам, которые решил отслеживать именно у этого ученика:

- какие правила добавлены ученику;
- какой уровень у ученика по каждому правилу;
- какие ошибки были замечены;
- что нужно повторить;
- какой комментарий оставить;
- что показать ученику.

Почти все сложные заранее заданные категории таджвида нужно убрать из MPMF 1.0. Вместо этого должен быть простой механизм выбора и добавления параметров.

## 2. Главный принцип

Преподаватель не должен заполнять большой шаблон таджвида на каждом уроке.

Он должен открыть ученика и видеть его текущие правила:

- правило;
- уровень ученика по этому правилу;
- ошибки или замечания;
- что нужно повторить;
- комментарий.

Если нужного правила нет у ученика, преподаватель должен иметь возможность добавить его прямо в карточке ученика или в блоке прогресса.

## 3. Что входит в MPMF 1.0

В MPMF 1.0 входит один простой блок прогресса.

Он может называться:

- "Прогресс";
- "Таджвид";
- "Чтение";
- "Ошибки и правила".

В блоке должны быть поля:

- правила таджвида, добавленные конкретному ученику;
- ошибки чтения;
- что нужно повторить;
- комментарий преподавателя;
- внутренний комментарий;
- видимость для ученика.

## 4. Уровень по правилу

В MPMF 1.0 уровень по правилу не заменяет оценку за урок.

Оценка за урок может быть 5-балльной или 10-балльной и относится к уроку в целом.

Уровень ставится по правилу, которое преподаватель добавил конкретному ученику.

Пример:

1. Преподаватель открыл ученика.
2. Добавил ученику правило "Сыфаты".
3. Поставил уровень ученика по этому правилу.
4. После следующих уроков обновляет этот уровень, если состояние изменилось.

Допустимые значения MPMF 1.0:

- `excellent` - отлично;
- `good` - хорошо;
- `satisfactory` - удовлетворительно;
- `poor` - плохо.

Этот уровень не является школьной балльной оценкой. Он показывает состояние конкретного правила у конкретного ученика.

Если у ученика добавлено несколько правил, у каждого правила может быть свой уровень.

Если преподаватель не хочет ставить уровень, он может просто добавить правило и оставить комментарий.

## 5. Правила таджвида

Правила таджвида в MPMF 1.0 не должны быть жестко зашиты большим списком.

Правила добавляются к конкретному ученику.

Преподаватель может:

- добавить одно или несколько правил ученику;
- поставить уровень по каждому правилу;
- изменить уровень по правилу;
- убрать правило из активного списка ученика, если оно больше не отслеживается;
- написать комментарий к правилу.

Примеры правил могут быть добавлены позже как стартовые варианты, но MPMF 1.0 не должен зависеть от полного заранее составленного списка.

Важно: если преподаватель хочет отслеживать у ученика "мадды", "нун сакина", "махрадж" или любое другое правило, он добавляет это правило именно ученику, а не включает большой раздел для всего курса.

## 6. Ошибки чтения

Ошибки чтения в MPMF 1.0 тоже могут фиксироваться по конкретному ученику.

Преподаватель может:

- выбрать одну или несколько ошибок у ученика;
- добавить новую ошибку ученику;
- указать, что ошибка повторяется;
- написать короткий комментарий;
- выбрать, показывать ли это ученику.

Ошибки не нужно заранее превращать в сложную систему категорий. Ошибки нужны как быстрые пометки, а не как обязательная часть каждого урока.

Пример:

- "ошибка в махрадже";
- "тянет не там";
- "путает правило";
- "нужно повторить";
- любой вариант, который добавит преподаватель.

Если в первом релизе нужно еще сильнее упростить интерфейс, ошибки можно оставить как текстовое поле или как тот же список параметров рядом с правилами.

## 7. Что нужно повторить

Поле "что нужно повторить" должно быть простым.

Варианты:

- текстовое поле;
- выбор из правил ученика;
- выбор из ошибок;
- комбинация выбора и текста.

Для MPMF 1.0 достаточно, чтобы преподаватель мог быстро написать или выбрать, что ученику повторить к следующему уроку.

## 8. Комментарии

Нужны два типа комментариев:

- комментарий для ученика;
- внутренний комментарий преподавателя.

Комментарий для ученика может быть показан в личном кабинете.

Внутренний комментарий видит только преподаватель и, если разрешено, админ для просмотра. Ученик его не видит.

## 9. Видимость для ученика

Преподаватель должен выбирать, что ученик видит.

Минимальные настройки:

- показать правила;
- показать ошибки;
- показать "что повторить";
- показать комментарий.

По умолчанию не нужно открывать ученику все внутренние заметки.

## 10. Где редактируются правила и ошибки

Правила и ошибки должны управляться в карточке ученика или в блоке прогресса ученика.

Допустимые варианты интерфейса MPMF 1.0:

- в карточке ученика: "Добавить правило" и "Добавить ошибку";
- в блоке прогресса ученика;
- из урока можно быстро перейти к прогрессу ученика, если преподаватель хочет обновить состояние после урока.

Главное: преподаватель не должен искать разные настройки по всему приложению.

Для MPMF 1.0 достаточно одного простого места:

- список правил конкретного ученика;
- список ошибок;
- добавление;
- редактирование названия;
- отключение/архивирование.

Когда ученик только создан, список правил у него может быть пустым.

Преподаватель добавляет ученику только те правила, которые хочет отслеживать.

Правила не нужно заранее добавлять всему курсу.

Удалять физически правила и ошибки не нужно, чтобы не ломать старые записи.

## 11. Связь с уроком и журналом

Прогресс в первую очередь связан с учеником.

Урок может быть поводом обновить прогресс ученика, но правила не являются заданием или темой урока.

В групповом уроке преподаватель может открыть конкретного ученика и обновить его прогресс.

В индивидуальном уроке преподаватель обновляет прогресс одного ученика.

В журнале месяца ячейка ученика может показывать:

- отсутствие, если ученик отсутствовал;
- краткий уровень по правилу ученика, если ученик присутствовал и уровень был поставлен;
- индикатор комментария;
- индикатор, что у ученика есть обновление прогресса.

Длинные списки правил и ошибок не должны отображаться прямо в ячейке журнала. Они открываются в карточке ученика или в его прогрессе.

## 12. Для кого функция

### 12.1. Преподаватель

Преподаватель:

- выбирает из правил, уже добавленных ученику, или добавляет новое правило этому ученику;
- выбирает ошибки;
- добавляет новые правила и ошибки;
- ставит уровень по правилу ученика;
- пишет комментарии;
- выбирает видимость для ученика.

### 12.2. Преподаватель-одиночка

Преподаватель-одиночка использует прогресс так же, как преподаватель.

Он сам управляет своим справочником правил и ошибок.

### 12.3. Администратор/директор

Администратор/директор может смотреть прогресс.

В MPMF 1.0 админ не редактирует учебные записи преподавателя.

Админ может видеть, что у ученика есть добавленные правила, ошибки, комментарий и уровни по правилам, если это нужно для контроля учебного процесса.

### 12.4. Ученик

Ученик видит только то, что преподаватель открыл:

- что повторить;
- комментарий;
- открытые правила;
- открытые ошибки.

Ученик не редактирует прогресс.

## 13. Пользовательский путь

### 13.1. Преподаватель фиксирует прогресс после урока

1. Преподаватель открывает урок или журнал.
2. Выбирает ученика.
3. Открывает прогресс ученика.
4. Видит список правил, которые уже отслеживаются у этого ученика.
5. При необходимости добавляет ученику новое правило.
6. По нужным правилам ставит или обновляет уровень: отлично, хорошо, удовлетворительно или плохо.
7. Выбирает ошибки, если их нужно зафиксировать.
8. Пишет, что нужно повторить.
9. Пишет комментарий для ученика или внутренний комментарий.
10. Выбирает видимость для ученика.
11. Сохраняет запись.

### 13.2. Ученик смотрит открытый прогресс

1. Ученик открывает кабинет.
2. Переходит в прогресс или видит краткий прогресс на главной.
3. Видит только открытые преподавателем данные.
4. Понимает, что нужно повторить.

## 14. Что не входит в MPMF 1.0

В MPMF 1.0 не входит:

- большой фиксированный шаблон всех разделов таджвида;
- обязательное заполнение махраджа, сыфатов, маддов и других разделов отдельными полями;
- сложная рубрика;
- автоматическая оценка чтения;
- проверка аудио;
- загрузка аудио;
- графики прогресса;
- подробная аналитика;
- обязательные баллы;
- сложная настройка критериев по каждому курсу.

## 15. Главное правило

Таджвид-прогресс должен быть максимально простым:

преподаватель открывает ученика, добавляет нужные правила и ошибки именно этому ученику, обновляет уровни и пишет короткий комментарий.

Все должно быть в одном понятном месте.
