# Feature Spec

# Homework

## 1. Назначение функции

Homework - это фиксация домашних заданий.

Функция нужна, чтобы преподаватель мог записать, что ученик или группа должны подготовить к следующему уроку.

В MPMF 1.0 ученик не сдает домашнее задание через систему. Домашнее задание нужно для фиксации: преподаватель указал, ученик увидел, дальше ученик сдает его вживую или на уроке.

## 2. Для кого функция

### 2.1. Преподаватель

Преподаватель создает домашние задания.

Он может:

- создать задание для группы;
- создать задание для отдельного ученика внутри группы;
- указать текст задания;
- указать срок;
- прикрепить материал;
- изменить или отменить задание;
- видеть историю заданий.

### 2.2. Преподаватель-одиночка

Преподаватель-одиночка использует домашние задания так же, как преподаватель.

### 2.3. Администратор / директор

Администратор/директор может смотреть домашние задания для контроля учебного процесса.

В MPMF 1.0 он не редактирует учебные записи преподавателя.

### 2.4. Ученик

Ученик видит свои домашние задания.

Он может:

- открыть задание;
- прочитать текст;
- открыть материалы;
- видеть срок.

Ученик не отправляет выполненную работу через систему в MPMF 1.0.

## 3. Главная идея

Домашнее задание должно быть быстрым текстовым действием в журнале.

Преподаватель не должен заполнять сложную форму. Основной сценарий: открыть урок, написать задание, при необходимости добавить материал или ссылку, сохранить.

## 4. Данные домашнего задания

Домашнее задание содержит:

- текст задания;
- курс;
- группу;
- учеников, которым оно назначено;
- дату создания;
- срок выполнения;
- автора;
- связанные материалы;
- статус видимости для ученика.

## 5. Задание для группы

Задание для группы получают все активные ученики группы.

Если нужно дать отдельное задание конкретному ученику, преподаватель может создать индивидуальное задание внутри группы.

## 6. Задание для индивидуального ученика

Задание для индивидуального ученика относится к его индивидуальному обучению.

Оно отображается в журнале индивидуального ученика и в личном кабинете ученика.

## 7. Индивидуальное задание внутри группы

В MPMF 1.0 преподаватель может задать отдельное задание конкретному ученику внутри группы.

Это нужно для случаев:

- ученик пропустил урок;
- ученик отстает;
- ученику нужно повторить отдельную тему;
- ученику дана индивидуальная цель.

## 8. Срок выполнения

У задания может быть срок.

Если срок не указан, задание считается актуальным до следующего урока или пока преподаватель не изменит его вручную.

В MPMF 1.0 не требуется сложная система просрочек по домашним заданиям.

## 9. Связь с материалами

К домашнему заданию можно прикрепить материал.

В MPMF 1.0 материалом может быть:

- текстовое описание;
- текстовый материал;
- ссылка.

Загрузка файлов и медиа не входит в MPMF 1.0.

## 10. Связь с журналом

Домашнее задание создается и отображается в календарном журнале.

В журнале должен быть виден признак, что к уроку есть домашнее задание.

## 11. Пользовательские пути

### 11.1. Преподаватель задает домашнее задание группе

1. Преподаватель открывает журнал группы.
2. Выбирает урок.
3. Пишет текст задания.
4. При необходимости добавляет материал.
5. Сохраняет.
6. Ученики группы видят задание в личном кабинете.

### 11.2. Преподаватель задает индивидуальное задание

1. Преподаватель открывает ученика.
2. Создает задание.
3. Указывает текст, срок и материалы.
4. Сохраняет.

### 11.3. Ученик смотрит домашнее задание

1. Ученик открывает личный кабинет.
2. Видит актуальные задания.
3. Открывает задание и материалы.

## 12. Проблемные ситуации

### 12.1. Преподаватель ошибся в задании

Преподаватель может изменить текст задания.

Система должна сохранять актуальную версию для ученика.

### 12.2. Задание отменено

Преподаватель может отменить задание или скрыть его из актуальных.

### 12.3. Ученик не видит задание

Преподаватель должен видеть, открыто ли задание ученику.

## 13. Права доступа

### 13.1. Преподаватель

Может создавать и редактировать свои задания.

### 13.2. Администратор / директор

Может смотреть задания для контроля.

### 13.3. Ученик

Может видеть только свои задания.

## 14. Что входит в MPMF 1.0

В MPMF 1.0 входит:

- домашнее задание для группы;
- индивидуальное задание ученику внутри группы;
- текст задания;
- срок;
- связь с материалами;
- отображение задания в журнале;
- отображение задания в личном кабинете ученика.

## 15. Что не входит в MPMF 1.0

В MPMF 1.0 не входит:

- сдача домашнего задания через систему;
- автоматическая проверка;
- оценки за домашнее задание как отдельный модуль;
- загрузка файлов;
- аудиоответы;
- чат по заданию;
- автоматические напоминания.

## 16. Открытые вопросы для следующих спецификаций

- Нужна ли история изменений задания.
- Нужен ли статус `выполнено` после MPMF 1.0.
- Нужна ли отметка преподавателя о проверке задания.
