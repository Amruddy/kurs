# Feature Spec

# Group Management

## 1. Назначение функции

Group Management - это управление учебными группами.

Функция нужна, чтобы администратор, директор или преподаватель-одиночка могли создать группу, связать ее с курсом, назначить преподавателя, добавить учеников и задать расписание.

Группа является основным учебным объектом для коллективных занятий.

## 2. Для кого функция

### 2.1. Администратор / директор

Администратор/директор управляет группами школы.

Он может:

- создать группу;
- выбрать курс;
- назначить преподавателя;
- добавить учеников;
- изменить состав группы;
- задать или изменить расписание;
- изменить статус группы;
- открыть журнал группы;
- видеть оплату по группе.

### 2.2. Преподаватель

Преподаватель работает с назначенными ему группами.

Он может:

- видеть свои группы;
- открыть группу;
- видеть список учеников;
- открыть журнал группы;
- вести посещаемость;
- вести прогресс;
- задавать домашние задания;
- видеть учебные материалы;
- видеть финансовые статусы учеников, если это разрешено школой.

### 2.3. Преподаватель-одиночка

Преподаватель-одиночка создает и ведет группы сам.

Он совмещает действия администратора и преподавателя.

### 2.4. Ученик

Ученик видит группу, в которой он учится.

Он не редактирует группу и не видит административные данные других учеников.

## 3. Главная идея

Группа должна быстро отвечать на вопросы:

- какой это курс;
- кто преподаватель;
- кто учится;
- когда проходят занятия;
- где журнал;
- есть ли проблемы с посещаемостью, прогрессом или оплатой.

## 4. Данные группы в MPMF 1.0

Группа содержит:

- название;
- курс;
- преподавателя;
- список учеников;
- расписание;
- статус;
- журнал;
- материалы группы;
- финансовые статусы учеников.

## 5. Название группы

Название группы должно быть понятным для администратора, преподавателя и ученика.

Примеры:

- `Таджвид, начинающие, воскресенье`;
- `Группа 1, дети`;
- `Вечерняя группа`.

В MPMF 1.0 система не обязана автоматически генерировать название группы.

## 6. Статусы группы

В MPMF 1.0 у группы есть статусы:

- активная;
- приостановлена;
- завершена;
- архивная.

Активная группа отображается в рабочих списках.

Приостановленная группа временно не проводит занятия.

Завершенная группа закончила обучение.

Архивная группа скрыта из основной работы, но доступна для истории.

## 7. Расписание группы

Группа может иметь повторяющееся расписание.

В MPMF 1.0 расписание должно поддерживать:

- дни недели;
- время начала;
- длительность или ориентировочное время окончания;
- период действия расписания;
- изменение расписания на будущее.

Если расписание изменилось, история прошедших уроков не должна ломаться.

В MPMF 1.0 уроки по расписанию создаются на один календарный месяц. Повторное
создание уроков за тот же месяц не должно создавать дубликаты.

## 8. Состав группы

В группу можно добавить учеников.

Ученик может:

- быть добавлен в группу;
- быть удален из группы;
- быть архивирован в рамках группы;
- перейти в другую группу через административное действие.

В MPMF 1.0 не требуется сложная история переводов между группами, но прошлые учебные записи ученика должны сохраняться.

## 9. Связь с журналом

У каждой группы есть календарный журнал.

Журнал группы показывает:

- даты занятий;
- учеников группы;
- посещаемость;
- оценки или отметки урока;
- комментарии;
- прогресс;
- домашние задания;
- материалы.

## 10. Связь с оплатой

По группе можно видеть финансовые статусы учеников.

В MPMF 1.0 это ручной учет:

- оплачен;
- ожидает оплаты;
- просрочено;
- освобожден от оплаты.

Автоматические платежи не входят в MPMF 1.0.

## 11. Пользовательские пути

### 11.1. Администратор создает группу

1. Администратор открывает раздел групп.
2. Создает группу.
3. Выбирает курс.
4. Назначает преподавателя.
5. Добавляет учеников.
6. Задает расписание.
7. Сохраняет группу.

### 11.2. Преподаватель открывает группу

1. Преподаватель входит в систему.
2. Видит список своих групп.
3. Открывает нужную группу.
4. Переходит в журнал, список учеников, материалы или оплату.

### 11.3. Ученик смотрит группу

1. Ученик входит в личный кабинет.
2. Видит свою группу.
3. Видит курс, расписание, домашние задания и материалы.

## 12. Проблемные ситуации

### 12.1. У группы нет расписания

Система должна показывать группе признак проблемы.

Администратор или преподаватель-одиночка должен добавить расписание.

### 12.2. У группы нет преподавателя

Администратор должен видеть такую группу как проблемную.

Журнал может быть доступен для просмотра, но вести занятия без преподавателя нельзя.

### 12.3. Ученик выбыл

Ученик убирается из активного состава группы.

Его прошлые записи посещаемости, прогресса, домашних заданий и оплаты сохраняются.

### 12.4. Группа завершила обучение

Группа переводится в статус `завершена` или `архивная`.

Новые занятия не создаются, история остается доступной.

## 13. Права доступа

### 13.1. Администратор / директор

Может создавать и редактировать группы школы.

### 13.2. Преподаватель

Может работать с назначенными группами.

### 13.3. Преподаватель-одиночка

Может управлять своими группами.

### 13.4. Ученик

Может видеть только свои группы и учебные данные.

## 14. Что входит в MPMF 1.0

В MPMF 1.0 входит:

- создание группы;
- связь группы с курсом;
- назначение преподавателя;
- добавление учеников;
- базовое расписание;
- статусы группы;
- журнал группы;
- просмотр оплаты по группе;
- архивирование группы.

## 15. Что не входит в MPMF 1.0

В MPMF 1.0 не входит:

- сложная CRM-воронка набора в группы;
- автоматическое распределение учеников;
- массовый импорт групп;
- продвинутая аналитика группы;
- чат группы;
- видеозанятия.

## 16. Открытые вопросы для следующих спецификаций

- Нужны ли несколько преподавателей на одну группу после MPMF 1.0.
- Как подробно хранить историю перевода ученика между группами.
- Нужны ли разные расписания для разных периодов курса.
