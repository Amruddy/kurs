# Page Contracts

# Контракты страниц MPMF 1.0

## 1. Назначение

Этот документ является главным источником правды для реализации страниц.

Для каждой страницы фиксируются:

- цель;
- основные блоки;
- кнопки;
- формы;
- данные;
- пустые состояния;
- границы реализации.

## 2. `/admin/groups`

Цель: список групп и быстрый переход к карточке группы.

Показывает:

- название группы;
- курс;
- преподавателя;
- количество активных учеников;
- краткое расписание;
- ближайшее занятие;
- статус;
- проблемные признаки.

Кнопки:

- `Создать группу`;
- `Открыть группу`.

Форма `Создать группу`:

- название;
- курс;
- преподаватель;
- статус.

Не делает:

- не показывает длинные формы расписания;
- не ведет журнал;
- не редактирует занятия.

## 3. `/admin/groups/[groupId]`

Цель: рабочая карточка группы для администратора.

Показывает:

- название группы;
- курс;
- преподавателя;
- статус;
- состав учеников;
- расписание;
- ближайшие занятия;
- ссылку на журнал для просмотра;
- оплату учеников группы;
- список незаполненных шагов: нет преподавателя, нет учеников, нет расписания,
  нет ближайших занятий.

Кнопки:

- `Изменить группу`;
- `Добавить ученика`;
- `Настроить расписание`;
- `Создать занятия`;
- `Открыть журнал`;
- `Открыть оплату`.

Форма `Настроить расписание`:

- день недели;
- время начала;
- время окончания;
- дата начала действия;
- дата окончания действия, если известна.

Действие `Создать занятия`:

- создает ближайшие занятия по активным правилам расписания;
- не создает дубликаты;
- не меняет прошедшие занятия.

Администратор на этой странице не редактирует учебные записи журнала.

## 4. `/teacher/groups`

Цель: список групп преподавателя.

Показывает:

- группы преподавателя;
- курс;
- количество учеников;
- ближайшее занятие;
- краткие проблемы.

Кнопки:

- `Открыть группу`;
- `Открыть журнал`;
- `Открыть ближайший урок`.

## 5. `/teacher/groups/[groupId]`

Цель: рабочая страница группы преподавателя.

Показывает:

- данные группы;
- расписание;
- ближайший урок;
- список учеников;
- последние занятия;
- последние домашние задания;
- доступные материалы;
- краткие сигналы по оплате, если преподавателю разрешено их видеть.

Кнопки:

- `Открыть ближайший урок`;
- `Открыть журнал`;
- `Задать домашнее задание`;
- `Добавить материал`;
- `Открыть ученика`.

## 6. `/teacher/groups/[groupId]/journal`

Цель: календарный журнал группы.

Показывает:

- выбранный месяц;
- даты занятий только этого месяца;
- строки учеников;
- ячейки посещаемости, оценки, комментария или прогресса;
- быстрый переход к уроку.

Кнопки:

- `Предыдущий месяц`;
- `Следующий месяц`;
- `Открыть урок`;
- быстрые отметки `П`, `Н`, `У`, если это удобно на текущем экране.

Правило отображения:

- таблица не показывает все дни месяца;
- колонки есть только у реально созданных занятий;
- на мобильном допускается режим: выбрать урок, затем работать со списком
  учеников.

## 7. `/teacher/lessons/[lessonId]`

Цель: подробное ведение конкретного урока.

Показывает:

- дату и время;
- курс и группу;
- преподавателя;
- тему;
- список учеников;
- отметки посещаемости;
- оценки за урок, если курс использует шкалу;
- комментарии;
- блок домашнего задания;
- блок материалов;
- переход к прогрессу ученика.

Кнопки:

- `Сохранить урок`;
- `Добавить домашнее задание`;
- `Добавить материал`;
- `Открыть прогресс ученика`.

Формы:

- тема урока;
- комментарий урока;
- отметки посещаемости по ученикам;
- домашнее задание;
- материал.

В версии 1.0 урок не получает отдельный пользовательский жизненный цикл
`начат/завершен`, пока отдельная спека не закрепит такое поведение.

## 8. `/student/schedule`

Цель: расписание ученика.

Показывает:

- ближайшие занятия;
- дату;
- время;
- курс;
- группу;
- преподавателя.

Пустое состояние:

- если занятий нет, показать, что расписание еще не создано или ближайших
  занятий нет.

Ученик не редактирует расписание.

## 9. Общие правила

- Страница списка не должна превращаться в карточку сущности.
- Карточка сущности содержит подробные действия.
- Журнал редактирует преподаватель.
- Администратор смотрит журнал для контроля.
- Ученик видит только свои данные.
- Любая кнопка должна вести к реально существующему действию.
