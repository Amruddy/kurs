# MPMF 1.0 Scope

# Состав версии 1.0

## 1. Назначение

Этот документ фиксирует состав первой полноценной рабочей версии `MPMF 1.0`.

Версия 1.0 должна быть достаточно цельной, чтобы администратор, преподаватель и
ученик могли пройти основной учебный путь без прямой работы в Supabase.

## 2. Обязательная основа

- Supabase-only база данных;
- dev-вход через seed-пользователей до отдельного этапа auth;
- роли `admin`, `teacher`, `student`;
- организация через `organization_id`;
- курсы таджвида;
- группы;
- ученики;
- назначение преподавателя на группу;
- назначение ученика в группу;
- расписание группы;
- создание занятий;
- журнал группы;
- страница урока;
- посещаемость;
- прогресс таджвида;
- домашние задания;
- материалы;
- ручная оплата;
- кабинет ученика.

## 3. Обязательные маршруты

Администратор:

- `/admin`;
- `/admin/courses`;
- `/admin/courses/[courseId]`;
- `/admin/groups`;
- `/admin/groups/[groupId]`;
- `/admin/students`;
- `/admin/students/[studentId]`;
- `/admin/teachers`;
- `/admin/payments`.

Преподаватель:

- `/teacher`;
- `/teacher/groups`;
- `/teacher/groups/[groupId]`;
- `/teacher/groups/[groupId]/journal`;
- `/teacher/lessons/[lessonId]`;
- `/teacher/students`;
- `/teacher/students/[studentId]`;
- `/teacher/attendance`;
- `/teacher/homework`;
- `/teacher/materials`;
- `/teacher/payments`.

Ученик:

- `/student`;
- `/student/schedule`;
- `/student/homework`;
- `/student/materials`;
- `/student/progress`;
- `/student/attendance`;
- `/student/payments`.

Общие:

- `/`;
- `/login`;
- `/forbidden`.

## 4. Порядок реализации после переписывания спеков

1. Админская карточка группы.
2. Расписание группы и создание занятий.
3. Преподавательская группа.
4. Журнал группы.
5. Страница урока.
6. Кабинет ученика: расписание.
7. Домашние задания и материалы.
8. Прогресс таджвида.
9. Оплата.

## 5. Не входит в версию 1.0

- онлайн-оплата;
- автоматические уведомления;
- чат;
- видеозвонки;
- загрузка файлов;
- мобильное приложение;
- родительский кабинет;
- сложная аналитика;
- публичный лендинг.
