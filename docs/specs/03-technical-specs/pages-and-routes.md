# Pages and Routes

# Страницы и навигация

## 1. Назначение документа

Этот документ описывает страницы MVP и основные переходы между ними.

Он не описывает визуальный дизайн, компоненты или API. Его задача - зафиксировать:

- какие рабочие области есть в системе;
- какие страницы нужны каждой роли;
- какие данные показывает каждая страница;
- какие основные действия доступны со страницы;
- как пользователь переходит между разделами.

## 2. Общий принцип навигации

После входа пользователь попадает в рабочую область, которая соответствует его
роли и правам.

Один пользователь может иметь несколько ролей. В таком случае он может
переключаться между доступными рабочими областями.

Базовые рабочие области MVP:

- преподаватель;
- ученик;
- администратор/директор;
- преподаватель-одиночка.

Преподаватель-одиночка технически использует рабочую область преподавателя и
упрощенную административную область своих данных.

## 3. Вход в систему

### 3.1. `/login`

Страница входа.

Показывает:

- кнопки dev-входа для Stage 0 и Stage 1:
  - "Войти как админ";
  - "Войти как преподаватель";
  - "Войти как ученик";
- поле логина и поле пароля позже, когда dev-вход заменяется настоящей
  авторизацией;
- ошибку входа, если выбранный тестовый пользователь недоступен или данные
  неверные.

После успешного dev-входа система выбирает соответствующего seed-пользователя,
определяет его роли и отправляет пользователя в подходящую рабочую область:

- админ попадает в `/admin`;
- преподаватель попадает в `/teacher`;
- ученик попадает в `/student`.

Dev-вход нужен только для разработки и ручной проверки MVP. Полноценная
регистрация, восстановление пароля и приглашения пользователей не входят в
Stage 0 и Stage 1.

### 3.2. Выбор рабочей области

Если у пользователя несколько ролей, система может показать выбор:

- войти как преподаватель;
- войти как ученик;
- войти как администратор/директор.

В MVP это может быть отдельная страница или переключатель внутри интерфейса.

## 4. Рабочая область преподавателя

Базовый префикс маршрутов:

- `/teacher`

### 4.1. `/teacher`

Главный экран преподавателя.

Показывает:

- ближайшие занятия;
- свои курсы;
- свои группы;
- список учеников, которым нужно напомнить об оплате;
- краткие проблемные признаки: пропуски, слабый прогресс, неоплата.

Основные действия:

- открыть группу;
- открыть журнал;
- открыть ближайший урок;
- открыть список оплат.

### 4.2. `/teacher/courses`

Расширенная страница после базового MVP.

Список курсов преподавателя.

Показывает:

- курсы, доступные преподавателю;
- тип курса;
- группы и индивидуальных учеников по курсу;
- включен ли таджвид-прогресс для курса.

Основные действия:

- открыть курс;
- открыть учеников курса и их прогресс.

Преподаватель школы не редактирует курс без отдельного права.

### 4.3. `/teacher/courses/:courseId`

Расширенная страница после базового MVP.

Страница курса.

Показывает:

- название курса;
- описание;
- формат обучения;
- группы курса;
- индивидуальных учеников курса;
- доступность таджвид-прогресса;
- материалы курса.

Основные действия:

- открыть группу;
- открыть индивидуального ученика;
- открыть материалы курса;
- открыть прогресс конкретного ученика.

### 4.4. `/teacher/groups`

Список групп преподавателя.

Показывает:

- группы преподавателя;
- курс каждой группы;
- статус группы;
- ближайшее занятие;
- количество учеников;
- краткие признаки проблем.

Основные действия:

- открыть группу;
- открыть журнал группы;
- открыть ближайший урок.

### 4.5. `/teacher/groups/:groupId`

Страница группы.

Показывает:

- название группы;
- курс;
- преподавателя;
- статус;
- расписание;
- список учеников;
- ссылку на журнал;
- домашние задания;
- материалы;
- финансовые статусы учеников, если у преподавателя есть доступ.

Основные действия:

- открыть журнал;
- открыть ученика;
- открыть урок;
- задать домашнее задание;
- добавить материал;
- посмотреть оплату.

### 4.6. `/teacher/groups/:groupId/journal`

Календарный журнал группы.

Показывает:

- месяц;
- занятия месяца;
- список учеников;
- отметки по датам занятий;
- статус проведения уроков;
- краткие отметки посещаемости, оценки, прогресса или комментариев.

Журнал показывает отметки по конкретным урокам. Сводная посещаемость по
ученикам и процентам находится в отдельном разделе `/teacher/attendance`.

Основные действия:

- выбрать месяц;
- открыть урок;
- начать урок;
- быстро отметить отсутствие;
- открыть запись ученика по уроку;
- сохранить изменения журнала.

### 4.7. `/teacher/lessons/:lessonId`

Страница урока.

Показывает:

- дату и время урока;
- группу;
- тему;
- статус проведения урока;
- список учеников, если урок групповой;
- отметки посещаемости;
- комментарии;
- быстрый переход к прогрессу выбранного ученика;
- домашнее задание;
- материалы.

Основные действия:

- изменить тему;
- начать урок;
- отметить посещаемость;
- оставить комментарий;
- открыть ученика и обновить его правила, ошибки или комментарий прогресса;
- задать домашнее задание;
- прикрепить материал;
- сохранить урок;
- завершить урок.

### 4.8. `/teacher/students`

Список учеников преподавателя.

Показывает:

- учеников из групп преподавателя;
- статус обучения;
- ближайший урок;
- признаки пропусков, слабого прогресса и неоплаты.

Основные действия:

- открыть карточку ученика;
- открыть журнал ученика;
- открыть прогресс;
- открыть оплату.

### 4.9. `/teacher/students/:studentId`

Карточка ученика.

Показывает:

- имя и контактные данные;
- активные группы;
- индивидуальное обучение;
- расписание;
- посещаемость;
- прогресс;
- домашние задания;
- материалы;
- оплату, если у преподавателя есть доступ.

Основные действия:

- открыть прогресс;
- открыть историю посещаемости;
- открыть домашние задания;
- открыть оплату;
- открыть связанный журнал.

### 4.10. `/teacher/attendance`

Сводная посещаемость учеников преподавателя.

Показывает:

- список учеников преподавателя;
- группу или курс ученика;
- процент посещаемости за выбранный период;
- количество проведенных уроков за период;
- количество посещенных уроков;
- количество пропущенных уроков;
- количество уважительных причин;
- дату последнего урока;
- короткий статус: `100%`, `есть пропуски`, `частые пропуски`, `нет данных`.

Основные фильтры:

- месяц;
- группа;
- курс;
- только ученики с пропусками;
- только ученики с низкой посещаемостью.

Основные действия:

- открыть ученика;
- открыть журнал группы;
- открыть последний урок;
- открыть историю посещаемости ученика.

### 4.11. `/teacher/individual`

Расширенная страница после базового MVP.

Список индивидуальных учеников.

Показывает:

- индивидуальных учеников преподавателя;
- курс;
- статус;
- расписание;
- ближайший урок;
- оплату, если доступна.

Основные действия:

- открыть индивидуального ученика;
- открыть индивидуальный журнал;
- открыть ближайший урок.

### 4.12. `/teacher/individual/:enrollmentId`

Расширенная страница после базового MVP.

Страница индивидуального обучения.

Показывает:

- ученика;
- курс;
- преподавателя;
- статус;
- расписание;
- журнал одного ученика;
- прогресс;
- домашние задания;
- материалы;
- оплату.

Основные действия:

- открыть урок;
- добавить занятие вручную;
- задать домашнее задание;
- заполнить прогресс;
- открыть оплату.

### 4.13. `/teacher/homework`

Список домашних заданий преподавателя.

Показывает:

- задания по группам;
- индивидуальные задания;
- сроки;
- связанные уроки;
- материалы.

Основные действия:

- создать задание;
- открыть задание;
- изменить задание;
- отменить или архивировать задание.

### 4.14. `/teacher/materials`

Материалы преподавателя.

Показывает:

- материалы курсов;
- материалы групп;
- материалы уроков;
- материалы домашних заданий;
- статус видимости.

Основные действия:

- добавить текстовый материал;
- добавить ссылку;
- скрыть материал;
- архивировать материал.

### 4.15. `/teacher/payments`

Оплата учеников преподавателя.

Показывает:

- оплату своих учеников;
- сумму;
- срок;
- период;
- статус;
- список неоплативших.

Основные действия:

- фильтровать неоплативших;
- открыть оплату ученика;
- изменить оплату, если есть `payments:write`.

## 5. Рабочая область ученика

Базовый префикс маршрутов:

- `/student`

### 5.1. `/student`

Главный экран ученика.

Показывает:

- ближайший урок;
- текущие курсы и группы;
- домашние задания;
- материалы;
- краткий прогресс;
- статус оплаты.

Основные действия:

- открыть ближайший урок;
- открыть домашнее задание;
- открыть материалы;
- открыть прогресс;
- открыть оплату.

### 5.2. `/student/schedule`

Расписание ученика.

Показывает:

- ближайшие занятия;
- дату и время;
- курс;
- группу;
- преподавателя.

### 5.3. `/student/courses/:courseId`

Расширенная страница после базового MVP.

Страница курса ученика.

Показывает:

- название курса;
- группу или индивидуальное обучение;
- преподавателя;
- ближайшие занятия;
- домашние задания;
- материалы;
- открытый прогресс;
- оплату по курсу.

### 5.4. `/student/homework`

Домашние задания ученика.

Показывает:

- активные задания;
- текст задания;
- срок;
- связанный урок;
- материалы;
- комментарий преподавателя.

Ученик не отправляет выполненную работу через систему в MVP.

### 5.5. `/student/materials`

Материалы ученика.

Показывает только открытые ученику материалы:

- материалы курса;
- материалы группы;
- материалы урока;
- материалы домашнего задания.

### 5.6. `/student/progress`

Прогресс ученика.

Показывает:

- открытые правила ученика и уровни по ним;
- открытые ошибки ученика;
- открытые комментарии;
- что нужно повторить;
- общую картину продвижения.

Не показывает внутренние заметки преподавателя.

### 5.7. `/student/attendance`

Посещаемость ученика.

Показывает:

- историю занятий;
- отсутствия;
- уважительные причины, если они открыты ученику;
- комментарии, если они открыты ученику.

Эта страница является подробной историей посещаемости ученика. Краткая
посещаемость может показываться на главном экране ученика. Ученик видит только
свои данные.

### 5.8. `/student/payments`

Оплата ученика.

Показывает:

- сумму ближайшей оплаты;
- срок;
- период;
- статус;
- задолженность.

Ученик не редактирует оплату.

## 7. Преподаватель-одиночка

Преподаватель-одиночка использует:

- `/teacher` для ведения уроков, журнала, прогресса, заданий и материалов;
- упрощенную административную область для создания курсов, групп, учеников,
  расписания и оплаты.

В MVP можно использовать тот же префикс `/admin`, но показывать только данные
его организации и скрывать школьные функции, которые ему не нужны.

## 8. Общие страницы ошибок и пустых состояний

### 8.1. Нет доступа

Если пользователь открывает страницу без прав, система показывает страницу
доступа запрещен.

Примеры:

- ученик пытается открыть админ-панель;
- преподаватель пытается открыть чужую группу;
- преподаватель без `payments:write` пытается изменить оплату.

### 8.2. Объект не найден

Если объект не существует или не относится к организации пользователя, система
показывает состояние "не найдено".

Для безопасности не нужно раскрывать, существует ли объект в другой
организации.

### 8.3. Пустые состояния

Пустые состояния нужны для:

- нет курсов;
- нет групп;
- нет учеников;
- нет ближайших занятий;
- нет домашних заданий;
- нет материалов;
- оплата не настроена;
- прогресс еще не открыт ученику.

Пустое состояние должно объяснять, что данных пока нет, и показывать действие,
если пользователь имеет право это действие выполнить.
