# Карта спецификаций проекта

Этот каталог хранит документы проекта от идеи к разработке.

## Порядок работы

1. `00-global-spec.md` - общее понимание проекта: кто, где, зачем.
2. `01-functional-map.md` - карта возможностей пользователей без техники.
3. `02-feature-specs/` - отдельные спецификации по функциям.
4. `03-technical-specs/` - база данных, страницы, API, связи частей проекта.
5. `04-visual-rules.md` - стиль, цвета, компоненты, правила интерфейса.
6. `04-interface-by-role.md` - структура интерфейса для преподавателя, ученика, преподавателя-одиночки и администратора.
7. `05-user-stories.md` - реальные пользовательские истории и проверка путей.
8. `06-work-plans/` - checklist перед кодом, активный план разработки и история выполненных этапов.

## Главное правило

Сначала spec, потом plan, потом code.

Если чего-то нет в спецификации, это не разрабатывается. Если в спецификации неясно, сначала уточняем.

Перед переходом к технической реализации нужно пройти:

`06-work-plans/00-before-stage-0-checklist.md`

Текущий активный план разработки:

`06-work-plans/active/01-mvp-implementation-plan.md`
