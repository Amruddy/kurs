# Карта спецификаций проекта

Этот каталог хранит актуальные спецификации Deshar для этапа `MPMF 1.0`.

`MPMF` означает `Minimum Product-Market Fit`: первая полноценная рабочая
версия, которой могут пользоваться администратор, преподаватель и ученик.

## Порядок чтения

1. `00-global-spec.md` - назначение продукта и границы версии 1.0.
2. `01-functional-map.md` - роли, рабочие сценарии и продуктовые возможности.
3. `02-feature-specs/` - спецификации отдельных функций.
4. `03-technical-specs/mpmf-1-scope.md` - состав версии 1.0.
5. `03-technical-specs/page-contracts.md` - контракт страниц, блоков, кнопок и
   форм.
6. `03-technical-specs/` - модель данных, действия, маршруты, права и
   валидация.
7. `04-interface-by-role.md` и `04-visual-rules.md` - интерфейс по ролям и
   визуальные правила.
8. `05-user-stories.md` - проверяемые пользовательские истории.

## Главное правило

Сначала spec, потом plan, потом code.

Если страницы, кнопки, формы или действия нет в спецификациях, оно не
разрабатывается. Если спецификация неясна, сначала обновляется спецификация.
