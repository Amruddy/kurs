# Functional Map

# Карта возможностей MPMF 1.0

## 1. Назначение

Этот документ фиксирует, что пользователи могут делать в версии `MPMF 1.0`.
Технические детали находятся в `03-technical-specs/`.

## 2. Администратор/директор

Администратор управляет учебной структурой.

Он может:

- открыть административную панель;
- создать курс;
- создать группу;
- назначить преподавателя;
- добавить ученика;
- назначить ученика в группу;
- настроить расписание группы;
- создать ближайшие занятия по расписанию;
- открыть группу и увидеть состав, расписание, ближайший урок, журнал и оплату;
- открыть ученика и увидеть обучение, посещаемость, прогресс и оплату;
- вести ручной учет оплат;
- видеть проблемные группы: нет преподавателя, нет расписания, нет учеников,
  нет ближайших занятий.

Администратор смотрит журнал, но не редактирует учебные записи преподавателя.

## 3. Преподаватель

Преподаватель ведет ежедневную учебную работу.

Он может:

- видеть свои группы и ближайшие занятия;
- открыть группу;
- открыть журнал группы за месяц;
- открыть конкретный урок;
- отметить посещаемость;
- поставить оценку за урок, если курс использует шкалу;
- оставить комментарий;
- обновить прогресс таджвида конкретного ученика;
- задать домашнее задание;
- добавить текстовый материал или ссылку;
- видеть оплату своих учеников, если это разрешено.

## 4. Ученик

Ученик использует личный кабинет.

Он может:

- видеть свои группы и курсы;
- видеть ближайшие занятия;
- открыть расписание;
- видеть домашние задания;
- видеть открытые материалы;
- видеть открытую часть прогресса;
- видеть свою посещаемость;
- видеть оплату.

Ученик не редактирует учебные данные, журнал, прогресс, материалы или оплату.

## 5. Преподаватель-одиночка

Преподаватель-одиночка использует:

- административные действия для своих курсов, групп, учеников, расписания и
  оплаты;
- преподавательские действия для журнала, уроков, прогресса, заданий и
  материалов.

## 6. Главные рабочие сценарии

### 6.1. Создание учебной структуры

1. Создать курс.
2. Создать группу.
3. Назначить преподавателя.
4. Добавить учеников.
5. Задать расписание.
6. Создать ближайшие занятия.

### 6.2. Проведение урока

1. Преподаватель открывает ближайший урок.
2. Отмечает отсутствующих или уважительную причину.
3. Добавляет тему, комментарий или оценку.
4. При необходимости обновляет прогресс.
5. Добавляет домашнее задание и материал.
6. Сохраняет урок.

### 6.3. Кабинет ученика

1. Ученик входит в кабинет.
2. Видит ближайший урок.
3. Смотрит задание и материалы.
4. Смотрит посещаемость и открытый прогресс.
5. Смотрит оплату.
