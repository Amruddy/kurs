# Готовность MPMF 1.0

## 1. Статус

`MPMF 1.0` находится на финальном этапе `Stage 16. MPMF 1.0 Release Hardening`.

Цель Stage 16 - не добавить новые функции, а подтвердить, что первая рабочая версия продукта собрана как цельный поток для администратора, преподавателя и ученика.

## 2. Что входит в MPMF 1.0

- dev-вход через стабильные тестовые роли;
- Supabase-only data layer без Prisma/local DB;
- административная область для курсов, групп, учеников, преподавателей и оплат;
- рабочие страницы преподавателя: группы, журнал, урок, ученики, домашние задания, материалы и оплаты;
- кабинет ученика: расписание, домашние задания, материалы, прогресс, посещаемость и оплаты;
- ручной учет оплат без онлайн-платежей;
- cross-role smoke-check основных маршрутов;
- мобильная и desktop-полировка базовых рабочих экранов.

## 3. Известные ограничения MPMF 1.0

- Используется dev-auth: вход кнопками `Войти как админ`, `Войти как преподаватель`, `Войти как ученик`.
- Нет production auth: нет настоящих паролей, email-приглашений, восстановления доступа и logout-сценария production-уровня.
- Нет онлайн-оплаты и платежных интеграций.
- Нет загрузки файлов, PDF, аудио и видео.
- Нет чата, уведомлений и видеозвонков.
- Нет родительского кабинета.
- Нет production deploy, домена, backup-политики и мониторинга.
- Нет сложной аналитики и отчетов после базовых рабочих списков.

## 4. Supabase

Файлы для dev-проекта Supabase:

- `supabase/schema.sql` - SQL-схема MPMF 1.0;
- `supabase/seed.sql` - стабильные dev-данные и test users;
- `supabase/README.md` - порядок ручного применения в Supabase SQL Editor.

Seed содержит стабильные email для smoke-проверки:

- `admin@example.test`;
- `teacher@example.test`;
- `student@example.test`;
- `solo-teacher@example.test`.

## 6. Следующий большой блок

После merge Stage 16 `MPMF 1.0` считается завершенным.

Следующий roadmap нужно открывать отдельно. Первый логичный блок:

`Production Auth And Real Accounts`

Он должен начинаться со спецификаций и work plan, потому что настоящие аккаунты не входят в MPMF 1.0 и меняют модель входа, приглашений, ролей и связи пользователя с учеником/преподавателем.
