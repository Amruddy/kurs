# Work Plans

Эта папка хранит планы разработки и историю выполненных этапов.

## Смысл раздела

Work Plans отвечают на вопросы:

- что делаем сначала;
- какой первый маленький этап;
- как проверять результат;
- какие связи между частями проекта нужно учитывать;
- что фиксировать после проверки в Git;
- какой следующий план или этап.

Главный принцип:

разработка идет маленькими проверяемыми шагами.

Общий статус текущего и следующего этапа фиксируется в:

`../roadmap/README.md`

## Как читать

1. Сначала пройти checklist:

   `00-before-stage-0-checklist.md`

2. Затем читать указатель активного плана:

   `active/README.md`

3. После выполнения этапа фиксировать результат в:

   `completed/`

## Правила

- `active/` - текущий рабочий план, по которому идет разработка.
- `completed/` - история выполненных этапов и проверок.
- Выполненные планы не являются источником новых задач.
- Если активный план противоречит спецификациям, сначала исправляется spec или plan.
- Если этап завершен, его результат записывается в `completed/`, а активный план обновляется при необходимости.
