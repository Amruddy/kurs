# MPMF 1.0 Spec Rewrite Plan

# План переписывания спецификаций под MPMF 1.0

## 0. Статус

- Статус плана: проверки пройдены, ожидает команды на commit/push.
- Ветка: `specs/mpmf-v1-rewrite`.
- Предыдущий завершенный stage: `Admin CRUD Foundation`, PR #26.
- Текущий черновик расписания после последнего коммита удален.

## 1. Цель

Полностью обновить спецификации под этап `MPMF 1.0`, убрать старую терминологию
и зафиксировать page-by-page контракт перед дальнейшей разработкой.

## 2. Что входит

- заменить старую терминологию на `MPMF 1.0` и `версия 1.0`;
- обновить карту спецификаций;
- переписать глобальную спецификацию и функциональную карту;
- заменить scope-файл на `mpmf-1-scope.md`;
- добавить `page-contracts.md`;
- обновить roadmap и рабочие планы;
- перенести завершенные планы из `active` в `completed`;
- не менять продуктовый код, кроме пользовательского текста, который содержит
  старую терминологию.

## 3. Что не входит

- реализация страниц;
- новые действия Supabase;
- изменение SQL-схемы;
- изменение текущей логики приложения.

## 4. Проверка готовности

Stage считается готовым, если:

- в документации и пользовательском тексте нет смысловых упоминаний старого
  термина;
- `docs/specs/README.md` указывает на новые источники правды;
- `docs/specs/03-technical-specs/page-contracts.md` описывает первые страницы
  для реализации;
- `docs/roadmap/README.md` отражает текущую ветку и stage;
- `docs/work-plans/active/README.md` указывает на этот план;
- `npm.cmd run lint` проходит успешно.

## 5. Запись о выполнении

Выполнено 2026-05-18:

- черновые действия после последнего коммита удалены;
- спецификации переведены на терминологию `MPMF 1.0` и `версия 1.0`;
- `docs/specs/03-technical-specs/mpmf-1-scope.md` заменяет старый scope;
- добавлен `docs/specs/03-technical-specs/page-contracts.md`;

- roadmap и `AGENTS.md` указывают на новый активный план;
