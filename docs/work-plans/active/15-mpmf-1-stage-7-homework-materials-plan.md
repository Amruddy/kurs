## 1. Цель

Дать преподавателю рабочие разделы домашних заданий и материалов, а ученику -
read-only просмотр назначенных заданий и открытых материалов. Связанные блоки на
странице урока должны оставаться частью этого же сценария: преподаватель задает
домашнее задание или материал после урока, ученик видит только доступную ему
часть.

## 3. Что входит

- Реализовать `/teacher/homework`:
  - список заданий преподавателя по его активным группам и ученикам;
  - создание задания для группы;
  - создание индивидуального задания ученику внутри группы;
  - срок, описание, связанный урок, статус.
- Реализовать `/teacher/materials`:
  - список материалов преподавателя по его курсам, группам, урокам, ученикам и
    домашним заданиям;
  - создание текстового материала или ссылки;
  - выбор видимости `teacher_only` или `visible_to_students`;
  - скрытие и архивирование через статусы.
- Реализовать `/student/homework`:
  - read-only список активных заданий ученика;
  - показ текста, срока, группы/курса/урока и связанных открытых материалов.
- Реализовать `/student/materials`:
  - read-only список только открытых ученику материалов со статусом `active`;
  - показ текстового материала или ссылки.
- Довести связанные server actions и write-layer функции для создания заданий
  и материалов вне страницы конкретного урока.
- Сохранить права:
  - преподаватель работает только со своими группами и учениками;
  - ученик видит только свои задания и материалы;
  - скрытые и архивные материалы ученику не показываются.

## 4. Что не входит

- Загрузка файлов, PDF, аудио, видео и изображений.
- Сдача домашнего задания учеником.
- Автоматическая проверка домашнего задания.
- Комментарии ученика и чат по заданию.
- Полноценная библиотека материалов с поиском.
- Админский раздел материалов школы.
- Онлайн-уведомления и напоминания.

## 5. Технический план

1. Расширить `app/lib/data/supabase-read.ts` типами и reader-функциями для
   списков заданий/материалов преподавателя и ученика.
2. Добавить write-layer функции в `app/lib/data/teacher-write.ts`:
   - создание задания для группы или ученика;
   - создание материала для курса, группы, урока, ученика или задания;
   - обновление статуса задания;
   - обновление статуса материала.
3. Подключить действия в `app/teacher/actions.ts` и revalidate нужных страниц.
4. Заменить заглушки:
   - `app/teacher/homework/page.tsx`;
   - `app/teacher/materials/page.tsx`;
   - `app/student/homework/page.tsx`;
   - `app/student/materials/page.tsx`.
5. При необходимости добавить минимальные CSS-правила для списков и форм в
   существующей System A без нового дизайн-направления.

## 6. Проверка готовности

- Преподаватель открывает `/teacher/homework` и видит задания своих групп.
- Преподаватель может создать групповое задание.
- Преподаватель может создать индивидуальное задание ученику внутри своей
  группы.
- Преподаватель открывает `/teacher/materials` и видит материалы своих учебных
  контекстов.
- Преподаватель может создать текстовый материал и ссылку.
- Ученик открывает `/student/homework` и видит только свои активные задания.
- Ученик открывает `/student/materials` и видит только материалы
  `visible_to_students` со статусом `active`.
- Скрытый или teacher-only материал не показывается ученику.
- `npm.cmd run lint` проходит успешно.
- `npm.cmd run build` проходит успешно.
- Codex выполнил локальный smoke-check, если dev-сервер и Supabase доступны.

## 7. Ручной smoke-чеклист после автоматических проверок

- Войти как преподаватель.
- Открыть `/teacher/homework`.
- Создать задание для группы и отдельного ученика.
- Открыть `/teacher/materials`.
- Создать текстовый материал и материал-ссылку.
- Войти как ученик.
- Открыть `/student/homework` и проверить видимость заданий.
- Открыть `/student/materials` и проверить, что скрытые материалы не видны.
