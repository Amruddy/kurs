## 1. Цель

Реализовать простой учет прогресса таджвида: преподаватель открывает карточку
своего ученика, добавляет отслеживаемые правила и ошибки, ставит уровень,
пишет что повторить и выбирает видимость для ученика. Ученик видит только
открытую часть прогресса на `/student/progress`.

## 3. Что входит

- Реализовать `/teacher/students/[studentId]` для преподавателя.
- Проверять, что преподаватель видит и меняет только учеников своих групп в
  текущей организации.
- Показать карточку ученика: контакты, активные группы, уроки, домашние
  задания, материалы, оплату и прогресс.
- Дать формы:
  - добавить правило ученику;
  - обновить уровень, комментарий и видимость правила;
  - добавить ошибку ученику;
  - обновить комментарий и видимость ошибки;
  - добавить запись прогресса: что повторить, комментарий ученику, внутренний
    комментарий, видимость и опциональная связь с уроком.
- Реализовать read-only `/student/progress` с показом только открытых ученику
  правил, ошибок и записей прогресса.
- Сохранить быстрые переходы к прогрессу из страницы урока через уже
  существующие ссылки на карточку ученика.

## 4. Что не входит

- Большой универсальный шаблон таджвида.
- Автоматическая оценка чтения.
- Графики и сложная аналитика.
- Загрузка аудио или файлов.
- Редактирование прогресса учеником.
- Админское редактирование записей прогресса преподавателя.

## 5. Технический план

1. Расширить `app/lib/data/supabase-read.ts` чтением карточки ученика
   преподавателя и прогресса ученика.
2. Добавить write-layer функции в `app/lib/data/teacher-write.ts` для правил,
   ошибок и записей прогресса.
3. Подключить функции в `app/teacher/actions.ts`.
4. Заменить заглушку `/teacher/students/[studentId]` рабочей карточкой ученика.
5. Заменить заглушку `/student/progress` read-only страницей прогресса.
6. Добавить минимальные CSS-правила для блоков прогресса в System A.

## 6. Проверка готовности

- Преподаватель открывает карточку своего ученика.
- Чужой пользователь в workspace преподавателя не раскрывает карточку ученика.
- Преподаватель может добавить правило, ошибку и запись прогресса.
- Преподаватель может поменять уровень, комментарии и видимость.
- Ученик видит на `/student/progress` только записи с открытой видимостью.
- `npm.cmd run lint` проходит успешно.
- `npm.cmd run build` проходит успешно.
- Codex выполнил локальный smoke-check, если dev-сервер и Supabase доступны.

## 7. Ручной smoke-чеклист после автоматических проверок

- Войти как преподаватель.
- Открыть `/teacher/students`.
- Открыть ученика из списка.
- Добавить правило и поставить уровень.
- Добавить ошибку.
- Добавить запись прогресса с видимостью ученику.
- Войти как ученик и открыть `/student/progress`.
- Убедиться, что видны только открытые данные.
