# План Entry Page перед Stage 6

## 0. Статус

- Статус плана: реализация проверена, ожидает подтверждения на commit/push.
- Ветка: `feat/mpmf-1-entry-page`.
- Roadmap: `docs/roadmap/mpmf-1-implementation-roadmap.md`.
- Предыдущий завершенный stage: `MPMF 1.0 Stage 5. Lesson Page`, PR #34.

## 1. Цель

Сделать главную входную страницу проекта, чтобы при открытии `/` было понятно,
что такое Deshar, кому он нужен и куда входить: администратору, преподавателю,
ученику или преподавателю-одиночке.

## 2. Источники правды

Перед кодом прочитаны и используются:

- `docs/roadmap/README.md`;
- `docs/work-plans/active/README.md`;
- `docs/work-plans/completed/12-mpmf-1-stage-5-lesson-page-plan.md`;
- `docs/specs/03-technical-specs/pages-and-routes.md`;
- `docs/specs/04-visual-rules.md`;
- текущие `app/page.tsx`, `app/login/page.tsx`, `app/components/app-shell.tsx`
  и `app/login/actions.ts`.

## 3. Что входит

- Заменить редирект `/ -> /login` на самостоятельный входной экран `/`.
- Сохранить `/login` как совместимый маршрут и показать на нем тот же входной
  экран.
- На входной странице показать:
  - название `Deshar`;
  - короткое рабочее объяснение продукта;
  - кому нужен продукт: школа/администратор, преподаватель, ученик,
    преподаватель-одиночка;
  - кнопки dev-входа по текущим seed-пользователям;
  - тестовые email для ручной проверки.
- Не добавлять публичный лендинг, регистрацию, оплату, каталог курсов или
  функции вне MPMF 1.0.

## 4. Технический план

1. Вынести входной экран в общий компонент для `/` и `/login`.
2. Обновить `app/page.tsx`, чтобы он рендерил входной экран вместо редиректа.
3. Обновить `/login`, чтобы использовать тот же компонент и сохранить обработку
   `error`.
4. При необходимости добавить CSS для компактных role cards в существующем
   System A.
5. Проверить lint, build и smoke главной страницы.

## 5. Проверка готовности

- `/` открывается без редиректа на `/login`.
- `/login` продолжает работать.
- На главной видно, что Deshar нужен школе/админу, преподавателю, ученику и
  преподавателю-одиночке.
- Кнопки dev-входа ведут в соответствующие рабочие области.
- Страница не выглядит как отдельный маркетинговый лендинг.
- `npm.cmd run lint` проходит успешно.
- `npm.cmd run build` проходит успешно.
