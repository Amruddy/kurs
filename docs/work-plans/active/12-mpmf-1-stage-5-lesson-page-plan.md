# План Stage 5. Lesson Page

## 0. Статус

- Статус плана: реализация проверена, ожидает подтверждения на commit/push.
- Ветка: `feat/mpmf-1-stage-5-lesson-page`.
- Roadmap: `docs/roadmap/mpmf-1-implementation-roadmap.md`.
- Предыдущий завершенный stage: `MPMF 1.0 Stage 4. Calendar Journal`, PR #33.

## 1. Цель

Реализовать рабочую страницу одного урока на маршруте
`/teacher/lessons/[lessonId]`, чтобы преподаватель мог открыть урок из журнала,
увидеть контекст занятия, заполнить учебную запись, отметить посещаемость,
поставить оценку за урок, добавить комментарии, домашнее задание и текстовый
материал или ссылку.

## 3. Что входит

- Реализовать `/teacher/lessons/[lessonId]` для преподавателя.
- Проверять, что преподаватель видит и меняет только свой урок в текущей
  организации.
- Показать дату, время, курс, группу, преподавателя, тему и учебную запись
  урока.
- Дать форму сохранения темы и общего комментария урока.
- Показать активных учеников группы.
- Дать форму журнала по ученикам:
  - посещаемость;
  - оценка за урок, если курс использует шкалу;
  - комментарий для ученика;
  - внутренний комментарий преподавателя;
  - видимость комментария ученику.
- Добавить быстрый переход к прогрессу ученика.
- Показать и добавить домашнее задание, связанное с уроком.
- Показать и добавить материал урока: текст или ссылку, с видимостью.
- Показать понятные пустые состояния для отсутствия учеников, домашки или
  материалов.

## 4. Что не входит

- Статусы урока `начат` / `завершен`.
- Полное редактирование прогресса ученика на странице урока.
- История изменений урока и журнала.
- Админское редактирование записей преподавателя.
- Загрузка файлов, PDF, аудио, видео или изображений.
- Автоматическая проверка домашнего задания.

## 5. Технический план

1. Расширить `app/lib/data/supabase-read.ts` чтением данных одного урока:
   урок, курс, группа, преподаватель, активные ученики, записи журнала,
   домашние задания, материалы и признаки прогресса.
2. Добавить чтение `lesson_mark_scale` курса для выбора оценок.
3. Расширить `app/lib/data/teacher-write.ts` действиями:
   - сохранение темы и учебной записи урока;
   - сохранение журнала по ученикам;
   - создание домашнего задания урока;
   - создание материала урока.
4. Подключить новые write-layer функции в `app/teacher/actions.ts`.
5. Заменить заглушку `/teacher/lessons/[lessonId]` рабочей страницей урока.
6. Добавить минимальные CSS-правила для формы урока и таблицы, сохранив
   существующий рабочий стиль System A.

## 6. Проверка готовности

Stage считается готовым, если:

- страница урока открывается преподавателю для своего урока;
- чужой пользователь в рабочей области преподавателя не раскрывает данные урока;
- видны дата, время, курс, группа, тема и текущая учебная запись;
- можно сохранить тему и общий комментарий урока;
- можно сохранить посещаемость, оценку и комментарии по ученикам;
- можно перейти к карточке ученика для прогресса;
- можно добавить домашнее задание урока;
- можно добавить текстовый материал или ссылку урока;
- `npm.cmd run lint` проходит успешно;
- `npm.cmd run build` проходит успешно;
- Codex выполнил локальный smoke-check, если dev-сервер и Supabase доступны.

## 7. Ручной smoke-чеклист после автоматических проверок

- Войти как преподаватель.
- Открыть `/teacher/groups/50000000-0000-4000-8000-000000000001/journal`.
- Открыть урок из колонки журнала.
- Изменить тему или комментарий урока и сохранить.
- Изменить посещаемость или оценку одного ученика и сохранить.
- Добавить домашнее задание.
- Добавить материал-ссылку.
- Обновить страницу и убедиться, что сохраненные данные видны.
- Проверить, что вход под другим пользователем не раскрывает этот урок.
