# План Stage 15. Mobile And UX Polish

## 0. Статус

- Статус плана: активная реализация.
- Ветка: `feat/mpmf-1-stage-15-mobile-ux-polish`.
- Roadmap: `docs/roadmap/mpmf-1-implementation-roadmap.md`.
- Предыдущий завершенный stage: `MPMF 1.0 Stage 14. Cross-role Smoke Flow`, смержен в `main`.

## 1. Цель

Довести основные рабочие экраны MPMF 1.0 до более пригодного состояния на desktop и mobile без добавления новых продуктовых функций и без смены дизайн-направления.

## 2. Источники правды

Перед кодом прочитаны и используются:

- `docs/roadmap/README.md`;
- `docs/roadmap/mpmf-1-implementation-roadmap.md`;
- `docs/work-plans/active/README.md`;
- `docs/work-plans/completed/22-mpmf-1-stage-14-cross-role-smoke-flow-plan.md`;
- `docs/specs/README.md`;
- `docs/specs/04-visual-rules.md`;
- `docs/specs/04-interface-by-role.md`;
- `docs/specs/03-technical-specs/page-contracts.md`.

## 3. Что входит

- Адаптивность основных рабочих страниц администратора, преподавателя и ученика.
- Более читаемые таблицы и рабочие списки на desktop и mobile.
- Улучшение мобильного поведения таблицы журнала: горизонтальная прокрутка, закрепленная колонка ученика и более крупные зоны нажатия.
- Более ровные кнопки, формы, раскрывающиеся панели и действия без визуальной сжатости.
- Согласование пустых состояний и информационных блоков с `04-visual-rules.md`.
- Проверка, что текст не перекрывается и не ломает кнопки/таблицы на узких экранах.
- Сохранение текущей функциональности и ролей без новых маршрутов.

## 4. Что не входит

- Новый дизайн-направление.
- Отдельное мобильное приложение.
- Новые продуктовые функции.
- Онлайн-оплата.
- Production auth.
- Новые зависимости только ради визуальной полировки.

## 5. Технический план

1. Обновить roadmap, active-plan, completed-plan и `AGENTS.md` со Stage 14 на Stage 15.
2. Проверить текущие общие компоненты, глобальные стили и ключевые страницы.
3. Внести общие CSS-улучшения для рабочих таблиц, форм, кнопок, списков, модальных/раскрывающихся панелей и mobile layout.
4. Точечно исправить страницы, где общий стиль не закрывает мобильную читаемость или журнал.
5. Запустить `npm.cmd run lint`, `npm.cmd run build` и `npm.cmd run smoke:roles`.
6. Выполнить локальный smoke ключевых страниц на dev-сервере.

## 6. Проверка готовности

- `npm.cmd run lint` проходит успешно.
- `npm.cmd run build` проходит успешно.
- `npm.cmd run smoke:roles` проходит успешно против локального dev-сервера.
- Основные страницы администратора, преподавателя и ученика открываются после полировки.
- Таблица журнала на мобильной ширине не ломает страницу и сохраняет доступ к именам учеников.
- Кнопки и формы читаются без визуальной сжатости.
- Раскрывающиеся панели и рабочие окна выглядят согласованно.
