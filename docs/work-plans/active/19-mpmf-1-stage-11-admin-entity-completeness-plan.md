# План Stage 11. Admin Entity Completeness

## 0. Статус

- Статус плана: активная реализация.
- Ветка: `feat/mpmf-1-stage-11-admin-entity-completeness`.
- Roadmap: `docs/roadmap/mpmf-1-implementation-roadmap.md`.
- Предыдущий завершенный stage: `MPMF 1.0 Stage 10. Payments`, смержен в `main` через PR #40.

## 1. Цель

Довести административные сущности до уровня MPMF 1.0: рабочие карточки курса и ученика, страницу преподавателей, базовое редактирование разрешенных полей, назначение учеников в группы и удаление из активного состава без физического удаления учебной истории.

## 3. Что входит

- `/admin/courses`:
  - переход из списка в карточку курса.
- `/admin/courses/[courseId]`:
  - данные курса, формат, статус, шкала оценок и настройки прогресса;
  - связанные группы, ученики, материалы и платежные сигналы;
  - базовое редактирование разрешенных полей;
  - архивирование курса без удаления учебной истории.
- `/admin/students`:
  - переход из списка в карточку ученика.
- `/admin/students/[studentId]`:
  - данные ученика, контакты, статус, группы, учебная история, прогресс, задания, материалы и оплаты;
  - базовое редактирование разрешенных полей;
  - назначение ученика в группу;
  - удаление ученика из активного состава группы через статус связи `removed` и `left_at`, без физического удаления истории;
  - архивирование ученика.
- `/admin/teachers`:
  - список преподавателей, контакты, назначенные группы, активные группы, ученики и ближайшие занятия;
  - создание/подключение преподавателя как пользователя с ролью `teacher`.

## 4. Что не входит

- Сложный HR-модуль преподавателей.
- Массовый импорт учеников.
- Юридическое архивирование и отчетность.
- Физическое удаление учебной истории.
- Новый публичный каталог курсов.

## 5. Технический план

1. Обновить roadmap, active-plan, completed-plan и `AGENTS.md` со Stage 10 на Stage 11.
2. Расширить read-layer для карточки курса, карточки ученика и списка преподавателей.
3. Реализовать write-layer для редактирования/архивации курса, редактирования/архивации ученика, создания преподавателя и мягкого удаления ученика из группы.
4. Заменить заглушки `/admin/courses/[courseId]`, `/admin/students/[studentId]`, `/admin/teachers` рабочими страницами.
5. Добавить переходы из списков курсов и учеников в карточки.
6. Выполнить автоматические проверки и локальный smoke-check.

## 6. Проверка готовности

- Администратор открывает `/admin/courses` и может перейти в карточку курса.
- Карточка курса показывает связанные группы, учеников, материалы, настройки и платежные сигналы.
- Администратор может изменить разрешенные поля курса и архивировать курс.
- Администратор открывает `/admin/students` и может перейти в карточку ученика.
- Карточка ученика показывает группы, историю уроков, прогресс, задания, материалы и оплаты.
- Администратор может изменить разрешенные поля ученика, назначить в группу и убрать из активного состава без удаления связи.
- `/admin/teachers` показывает реальных преподавателей и позволяет добавить преподавателя.
- `npm.cmd run lint` проходит успешно.
- `npm.cmd run build` проходит успешно.
- Codex выполняет локальный smoke-check, если dev-сервер и Supabase доступны.
