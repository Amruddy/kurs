# План Stage 4. Calendar Journal

## 0. Статус

- Статус плана: реализация и проверки выполнены, ожидает команды на commit/push.
- Ветка: `feat/mpmf-1-stage-4-calendar-journal`.

- Предыдущий завершенный stage: `MPMF 1.0 Stage 3. Teacher Groups`, PR #32.

## 1. Цель

Реализовать рабочий календарный журнал группы преподавателя на маршруте
`/teacher/groups/[groupId]/journal`: таблицу выбранного месяца только по
реальным урокам группы, со строками учеников, быстрыми отметками посещаемости,
краткими признаками учебной записи и переходами к урокам.

## 3. Что входит

- Реализовать `/teacher/groups/[groupId]/journal` для преподавателя.
- Проверять, что преподаватель видит только журнал группы, где он назначен
  преподавателем в текущей организации.
- Показать верхнюю часть с группой, курсом, преподавателем, выбранным месяцем
  и переходом обратно к группе.
- Добавить навигацию по месяцам.
- Показать таблицу журнала:
  - строки - активные ученики группы;
  - колонки - только реальные уроки выбранного месяца;
  - заголовок колонки - дата, день недели, время и ссылка на урок;
  - ячейка - короткая отметка посещаемости и признаки оценки, комментария,
    прогресса, домашнего задания или материала.
- Поддержать быстрые отметки посещаемости:
  - пусто;
  - `П`;
  - `Н`;
  - `У`.
- Сохранять отметки посещаемости через серверное действие преподавателя.
- Показать понятные пустые состояния, если в месяце нет уроков или в группе нет
  активных учеников.

## 4. Что не входит

- Админское редактирование журнала.
- Сводная аналитика по посещаемости или прогрессу.
- Колонки для дней месяца без уроков.
- Полная страница урока `/teacher/lessons/[lessonId]`.
- Редактирование темы, конспекта, домашнего задания, материалов, подробных
  комментариев и прогресса урока.
- Экспорт журнала.

## 5. Технический план

1. Расширить `app/lib/data/supabase-read.ts` чтением данных для календарного
   журнала группы преподавателя.
2. Добавить типы строк журнала, уроков месяца, учеников и ячеек.
3. Читать уроки только в границах выбранного месяца и только для выбранной
   группы текущего преподавателя.
4. Читать `journal_entries`, `progress_records`, `homework` и `materials` для
   признаков в ячейках.
5. Добавить write-layer для сохранения быстрых отметок посещаемости с проверкой
   принадлежности группы текущему преподавателю.
6. Реализовать `saveGroupJournal` в `app/teacher/actions.ts`.
7. Реализовать UI страницы журнала существующими рабочими паттернами проекта и
   CSS-классами таблицы журнала.

## 6. Проверка готовности

Stage считается готовым, если:

- `/teacher/groups/[groupId]/journal` открывается для своей группы
  преподавателя;
- чужая группа для преподавателя не раскрывает данные;
- таблица показывает только уроки выбранного месяца, а не все дни месяца;
- в первой колонке закреплен список учеников;
- колонки уроков ведут на `/teacher/lessons/[lessonId]`;
- быстрые отметки `П`, `Н`, `У` можно выбрать и сохранить;
- в ячейках видны краткие признаки оценки, комментария, прогресса, домашнего
  задания или материала, если такие данные есть;
- пустой месяц показывает спокойное пустое состояние;
- `npm.cmd run lint` проходит успешно;
- `npm.cmd run build` проходит успешно;
- Codex выполнил локальный smoke-check, если dev-сервер и Supabase доступны.

## 7. Ручной smoke-чеклист после автоматических проверок

- Войти как преподаватель.
- Открыть `/teacher/groups`.
- Открыть свою группу и перейти в журнал.
- Проверить, что журнал показывает выбранный месяц и только даты занятий.
- Проверить переходы по колонкам уроков.
- Изменить одну отметку посещаемости на `Н` или `У`, сохранить и обновить
  страницу.
- Убедиться, что отметка сохранилась.
- Убедиться, что чужая группа не раскрывается в рабочей области преподавателя.
