# План Stage 3. Teacher Groups

## 0. Статус

- Статус плана: завершен и смержен в `main` через PR #32.
- Ветка: `feat/mpmf-1-stage-3-teacher-groups`.
- Roadmap: `docs/roadmap/mpmf-1-implementation-roadmap.md`.
- Предыдущий завершенный stage: `MPMF 1.0 Stage 2. Group Schedule And Lessons`, PR #31.

## 1. Цель

Дать преподавателю рабочий вход в свои группы через маршруты `/teacher/groups`
и `/teacher/groups/[groupId]`: список назначенных групп, ближайший урок,
состав, последние занятия и быстрые переходы к журналу, ближайшему уроку и
ученику.

## 3. Что входит

- Обновить `/teacher/groups`, чтобы список групп преподавателя показывал:
  - группу;
  - курс;
  - статус;
  - количество активных учеников;
  - ближайшее занятие;
  - краткие проблемные признаки.
- Добавить действия из списка групп:
  - `Открыть группу`;
  - `Открыть журнал`;
  - `Открыть ближайший урок`, если ближайший урок есть.
- Реализовать `/teacher/groups/[groupId]` как рабочую страницу группы
  преподавателя.
- На странице группы показать:
  - данные группы;
  - курс;
  - преподавателя;
  - статус;
  - расписание;
  - ближайший урок;
  - список активных учеников;
  - последние занятия;
  - последние домашние задания;
  - доступные материалы;
  - краткие сигналы по оплате, если преподаватель видит оплату своих учеников.
- Добавить быстрые переходы:
  - к ближайшему уроку;
  - к журналу;
  - к ученику.
- Проверять, что преподаватель видит только группы, где он назначен
  преподавателем в текущей организации.
- Показать понятные пустые состояния для отсутствующих групп, занятий, учеников,
  домашних заданий, материалов и оплаты.

## 4. Что не входит

- Редактирование админских данных группы.
- Изменение состава группы.
- Настройка расписания преподавателем школы.
- Массовые действия журнала.
- Реализация календарного журнала `/teacher/groups/[groupId]/journal`.
- Реализация страницы урока `/teacher/lessons/[lessonId]`.
- Создание или редактирование домашнего задания и материалов сверх уже
  существующих заглушек/actions.

## 5. Технический план

1. Расширить `app/lib/data/supabase-read.ts` типами и чтением данных для
   детальной страницы группы преподавателя.
2. Усилить `getTeacherGroups`: вернуть ближайший урок как ссылку, список
   проблемных признаков и действия для строк.
3. Добавить `getTeacherGroupDetail(organizationId, email, groupId)` с проверкой,
   что группа принадлежит текущему преподавателю.
4. Для детальной страницы читать активный состав группы, активные правила
   расписания, ближайшие и последние занятия, активные домашние задания,
   активные материалы и оплаты учеников группы.
5. Обновить UI `/teacher/groups` и `/teacher/groups/[groupId]` существующими
   компонентами `SupabaseDataPage`, `DataTable`, `InfoList` и текущими
   CSS-паттернами.
6. Не добавлять write-actions в этот stage, кроме уже существующих переходов к
   будущим страницам.

## 6. Проверка готовности

Stage считается готовым, если:

- `/teacher/groups` показывает только группы текущего преподавателя;
- список групп показывает курс, статус, количество учеников и ближайший урок;
- из списка можно открыть группу, журнал и ближайший урок, если он есть;
- `/teacher/groups/[groupId]` открывается для своей группы преподавателя;
- чужая группа для преподавателя не раскрывает данные и показывает безопасную
  ошибку чтения;
- страница группы показывает расписание, ближайший урок, учеников, последние
  занятия, домашние задания, материалы и сигналы по оплате;
- на странице нет админского редактирования группы, состава или расписания;
- пустые состояния понятны и не выглядят ошибкой;
- `npm.cmd run lint` проходит успешно;
- `npm.cmd run build` проходит успешно;
- Codex выполнил локальный smoke-check, если dev-сервер и Supabase доступны.

## 7. Ручной smoke-чеклист после автоматических проверок

- Войти как преподаватель.
- Открыть `/teacher/groups`.
- Проверить, что видны только группы текущего преподавателя.
- Открыть группу из списка.
- Проверить, что на странице есть данные группы, расписание, ближайший урок,
  ученики и последние занятия.
- Проверить переходы `Открыть журнал`, `Открыть ближайший урок` и
  `Открыть ученика`.
- Убедиться, что на странице группы нет админских кнопок изменения группы,
  состава или расписания.
