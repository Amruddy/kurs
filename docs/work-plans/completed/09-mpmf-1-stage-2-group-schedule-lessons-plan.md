# План Stage 2. Group Schedule And Lessons

## 0. Статус

- Статус плана: реализация и проверки выполнены, ожидает команды на commit/push.
- Ветка: `feat/mpmf-1-stage-2-group-schedule-lessons`.

- Предыдущий завершенный stage: `MPMF 1.0 Stage 1. Admin Group Detail`, PR #30.

## 1. Цель

Дать администратору возможность настроить повторяющееся расписание группы и
создать ближайшие занятия по активным правилам на маршруте
`/admin/groups/[groupId]`.

## 3. Что входит

- Добавить на `/admin/groups/[groupId]` форму `Настроить расписание`.
- Создавать `ScheduleRule` для группы:
  - один или несколько дней недели за одно действие;
  - время начала;
  - время окончания;
  - дата начала действия;
  - дата окончания действия, если известна.
- Дать возможность физически удалить активное правило расписания.
- При удалении правила удалить будущие уроки, созданные этим правилом, если у
  них нет записей журнала.
- Перед удалением правила обязательно требовать второй клик: кнопка с иконкой
  корзины остается того же размера, становится красной и показывает маленькую
  подпись `Точно`.
- Показывать активное расписание в карточке группы как читаемые карточки:
  день недели, время и период действия.
- В форме и карточках расписания показывать неделю с понедельника.
- В верхней части карточки группы показывать действие `Настройки`, где собраны
  изменения группы, состава, расписания и создание уроков.
- Добавить действие `Создать уроки` с выбором окна создания:
  - на один месяц;
  - на три месяца;
  - до даты окончания расписания.
- Создавать `Lesson` по активным правилам расписания только в выбранном окне.
- Не создавать дубликаты занятий при повторном запуске.
- Не менять прошедшие занятия.
- Показывать ближайшие занятия и понятные пустые состояния.

## 4. Что не входит

- Перенос занятий.
- Отмена занятий.
- Исключения календаря и праздничные дни.
- Жизненный цикл урока `начат/завершен`.
- Редактирование журнала.
- Преподавательские страницы групп.

## 5. Технический план

1. Расширить `app/lib/data/admin-write.ts` действиями создания/удаления
   `ScheduleRule` и генерации `Lesson`.
2. Реализовать валидацию периода расписания и времени начала/окончания.
3. Генерировать занятия по активным правилам с учетом `starts_on`, `ends_on`,
   `weekday`, `start_time`, `end_time` и таймзоны `Europe/Moscow`.
4. Для окна `до даты окончания расписания` требовать заполненный `ends_on` у
   всех активных правил.
5. Защиту от дублей делать на уровне существующего уникального ключа
   `organization_id, schedule_rule_id, starts_at` и предварительной выборки.
6. Обновить `app/admin/actions.ts` для реальных actions:
   `createGroupScheduleRule`, `deleteScheduleRule`,
   `generateLessonsForGroup`.
7. Обновить UI `/admin/groups/[groupId]`: добавить кнопки и формы расписания,
   карточный список правил и кнопку создания занятий.
8. При необходимости расширить `AdminGroupDetailData`, чтобы UI видел
   технические поля правил расписания.

## 6. Проверка готовности

Stage считается готовым, если:

- админ может создать правило расписания в карточке группы;
- админ может выбрать несколько дней недели при создании правила;
- правило появляется в блоке расписания как читаемая карточка;
- дни недели в форме и блоке расписания идут с понедельника;
- правило можно удалить, и оно исчезает из активного расписания;
- перед удалением правила сама кнопка с иконкой корзины становится красной, не
  меняет размер и показывает маленькую подпись `Точно`;
- настройки группы собраны в верхнем действии `Настройки`;
- `Создать уроки` позволяет выбрать окно создания: один месяц, три месяца или
  до окончания расписания;
- `Создать уроки` создает занятия по активным правилам только в выбранном окне;
- повторный запуск `Создать уроки` не создает дубли;
- ближайшие занятия отображаются в карточке группы;
- `npm.cmd run lint` проходит успешно;
- `npm.cmd run build` проходит успешно;
- Codex выполнил локальный smoke-check, если dev-сервер и Supabase доступны.

## 7. Ручной smoke-чеклист после автоматических проверок

- Войти как админ.
- Открыть `/admin/groups`.
- Открыть существующую группу.
- Нажать `Настроить расписание`, выбрать несколько дней недели и добавить новое правило.
- Нажать `Создать уроки`, выбрать окно `На 1 месяц`.
- Проверить, что ближайшие занятия появились.
- Нажать `Создать уроки` повторно и убедиться, что дубли не появились.
- Если у всех правил заполнено `Действует до`, проверить окно `До окончания расписания`.
- Удалить правило расписания и убедиться, что оно исчезло из активного
  списка.
- Перед удалением убедиться, что кнопка с иконкой корзины становится красной,
  не увеличивается и показывает маленькую подпись `Точно`.
